Getting Started

Welcome to the HCP Anywhere File Sync and Share API Reference Guide

This is your guide to using the HCP Anywhere File Sync and Share API. The File Sync and Share API is a set of HTTP endpoints which allows you to integrate custom and third-party applications with HCP Anywhere.

Each of the File Sync and Share endpoints can be used to perform a different operation in HCP Anywhere. These operations include uploading and updating files, creating folders and links, deleting files and folders, and retrieving metadata about files and folders.

Note: In order to use the File Sync and Share API, the File Sync and Share API needs to be enabled in the Management Console by the HCP Anywhere administrator.

Authentication

To use the File Sync and Share API, you need to first authenticate with the system. This is done by submitting the username and password you use to access HCP Anywhere to the OAuth endpoint. If the request is successful, an access token is returned which holds your login credentials.

For information on requesting an access token, see the authentication endpoint.

Once you have an access token, you can use the other File Sync and Share API endpoints by supplying the access token and any other required parameters of the endpoint. The access token is supplied as a header in each request. In cURL, the authorization header looks like this:

-H "Authorization: Bearer abc="

In an actual request, the access token is much longer than the “abc=” access token provided here.

Request Format

In general, the File Sync and Share API uses HTTP POST requests with JSON arguments and JSON responses. In each request, there are four header parameters that need to be supplied:

  • Access token
  • Content type used in the request
  • Content type accepted as a response
  • File Sync and Share API version used in the request

Content type header

The content type header specifies the type of content that is used in a request. For most requests, this is application/json. However, there are a few exceptions to this. Basic authentication uses an application/x-www-form-urlencoded content header while both upload and update use an appliction/octet-stream content header.

For the endpoints that use JSON, the content type header looks like this in cURL:

-H "Content-Type: application/json"

Accept header

The accept header specifies the type of content that is accepted as a response from a request. For most requests, this is application/json. However, the download file endpoint does not accept application/json in a response. Instead, the download file endpoint accepts application/octet-stream in a response.

For the endpoints that accept JSON, the accept header looks like this in cURL:

-H "Accept: application/json"

Versioning header

The version header specifies what version of the File Sync and Share API you’re using.

The HCP Anywhere server is backwards compatible. That means that older versions of the File Sync and Share API can make requests to newer versions of the server. However, version 2.1.1 of HCP Anywhere is the first version of the File Sync and Share API and therefore the oldest version that can be used to make requests.

For version 2.1.1 of the File Sync and Share API, the version header looks like this in cURL:

-H "X-HCPAW-FSS-API-VERSION: 2.1.1"

URL Request Structure

For each File Sync and Share API endpoint, the URL has the same structure. The URL begins like this:

https://anywhere.example.com/fss/public

This is followed by the URI of whichever endpoint you are using. For example, the URL for an upload new file request looks like this in cURL:

https://anywhere.example.com/fss/public/file/stream/create

In an actual request, “anywhere.example.com” should be replaced with the server that the request is being made to.

Response Headers

For every successful request, this information is returned in the response header:

  • X-HCPAW-SUPPORTED-FSS-API-VERSIONS: The File Sync and Share API versions supported by the server
  • X-HCPAW-FSS-API-VERSION: The File Sync and Share API version used in the request
  • X-HCPAW-System-Id: The system id, which will always be the global unique identifier of the system

Additional Resources

In addition to this document, there is a Java software development kit (SDK) available in the Maven Central Repository. The SDK uses the File Sync and Share API to access HCP Anywhere. There is also a sample program that uses the SDK on GitHub. The sample program is just one example of how you might use File Sync and Share SDK.
