Get user metadata

Gets information about a user.

The access token provided in the authentication header identifies the user to get information about.

URI

/fss/public/user/get

HTTP Method

POST

Parameters

There are no parameters for a get user metadata request.

Example Request

Here is an example of a get user metadata request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/user/get -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1"

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to get the user metadata.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to get the user metadata. The request should be retried.
