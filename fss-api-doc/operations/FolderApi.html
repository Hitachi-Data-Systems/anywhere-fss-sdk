Folder

This page consists of the Folder endpoints for the HCP Anywhere File Sync and Share API.

For each endpoint, the following information is provided (if applicable):

  • Description of the operation the endpoint performs
  • URI for the endpoint
  • HTTP method for the endpoint
  • Parameters for the endpoint
  • Example request
  • Example response
  • Description of the response
  • Response codes and description of what each response code means

Create folder

Creates a folder in HCP Anywhere.

URI

/fss/public/folder/create

HTTP Method

POST

Parameters

  • Request Body:
    Request body for create folder▾

    Request body parameters needed to create a folder. This request body needs to be JSON.

Example Request

Here is an example of a create folder request in cURL. In this example, the folder being created is "Sales-Reports" and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/folder/create -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"path": "/Work-Projects/Sales-Reports", "createParents": true}'

Example Response

The response for a successful create folder request. In an actual response, the etag is much longer. 

 { 
 "type": "FOLDER", 
 "parent": "/Work-Projects", 
 "name": "Sales-Reports", 
 "changeTime": 1456243695357, 
 "access": "COLLABORATOR", 
 "etag": "kaf==" 
 }

Description of Response

Response body for create folder▾

Response Codes

  • 201 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to create the folder.
  • 404 - A parent of the given path does not exist, and createParents was set to false.
  • 409 - A conflict was encountered while trying to create the folder.
  • 413 - A limit on HCP Anywhere was encountered while trying to create the folder. The error message contained in the reponse body includes more information.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to create the folder. The request should be retried.

List folder contents

Lists the contents of a folder.

URI

/fss/public/folder/entries/list

HTTP Method

POST

Parameters

  • Request Body:
    Request body for list folder▾

    Request body parameters needed to list the contents of a folder. This request body needs to be JSON.

Example Request

Here is an example of a list folder request in cURL. In this example, the folder to list the contents of is "Sales-Reports" and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/folder/entries/list -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"path":“/Work-Projects/Sales-Reports"}'

Example Response

The response for a successful list folder request. In an actual response, the page token, hash, and etags are much longer. 

 {

 "access": "COLLABORATOR", 
 "etag": "wjq==" 
 }, 
 { 
 "type": "FILE",

 "changeTime": 1455813409187, 
 "size": 245914, 
 "hash": "93ad2", 
 "access": "COLLABORATOR", 
 "etag": "ljs==" 
 }, 
 ] 
 }

Description of Response

Response body for list folder▾

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to list the contents of the folder.
  • 404 - The given path does not exist.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to list the contents of the folder. The request should be retried.
