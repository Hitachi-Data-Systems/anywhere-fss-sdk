Path

This page consists of the Path endpoints for the HCP Anywhere File Sync and Share API.

For each endpoint, the following information is provided (if applicable):

  • Description of the operation the endpoint performs
  • URI for the endpoint
  • HTTP method for the endpoint
  • Parameters for the endpoint
  • Example request
  • Example response
  • Description of the response
  • Response codes and description of what each response code means

Delete

Deletes a file or folder at a given path in HCP Anywhere.

URI

/fss/public/path/delete

HTTP Method

POST

Parameters

  • Request Body:
    Request body for delete▾

    Request body parameters needed to delete a file or folder. This request body needs to be JSON.

Example Request

Here is an example of a delete request in cURL. In this example, the file to delete is "Sales-Presentation.docx" and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/path/delete -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"path": "/Work-Projects/Sales-Reports/Sales-Presentation.docx", "recursive": false, "etag": "ePM=="}'

Example Response

For a successful delete, no information is returned.

▾

Response Codes

  • 204 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to delete the entry.
  • 404 - The given path does not exist.
  • 409 - A conflict was encountered while trying to delete the entry.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to delete the file. The request should be retried.

Get file/folder metadata

Gets the metadata of a file or folder in HCP Anywhere.

URI

/fss/public/path/info/get

HTTP Method

POST

Parameters

  • Request Body:
    Request body for get metadata▾

    Request body parameters needed to get the metadata of a file or folder. This request body needs to be JSON.

Example Request

Here is an example of a get file metadata request. In this example, the file to get metadata about is "Sales-Presentations.docx" and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/path/info/get -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"path": "/Work-Projects/Sales-Reports/Sales-Presentation.docx"}'

Example Response

The response for a successful get metadata request. In an actual response, the etag and hash are much longer.

 "parent": "/Work Projects/Sales-Reports", 
 "name": "Sales-Presentation.docx", 
 "changeTime": 1456260358874, 
 "size": 16030, 
 "hash": "0dc6b", 
 "access": "COLLABORATOR", 
 "etag": "rbs==" 
 }

Description of Response

Response body for get metadata▾

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to get metadata about the entry.

Move/rename

Moves, renames, or both moves and renames a file or folder in HCP Anywhere.

When moving a folder, all the contents of the folder are also moved.

To rename a file or folder, enter a new name for the file or folder in the destinationPath parameter in the request body.

URI

/fss/public/path/move

HTTP Method

POST

Parameters

  • Request Body:
    Request body for move▾

    Request body parameters needed to move or rename a file or folder. This request body needs to be JSON.

Example Request

Here is an example of a move request in cURL. In this example, the file to move is "Q1-Notes.docx" and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/path/move -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"sourcePath": "/Q1-Notes.docx", "destinationPath": "/2016/Q1-Notes.docx", "etag": "ePM==", "createParents": true}'

Example Response

The response for a successful move request. In an actual response, the access token and hash are much longer. 

 { 
 "type":"FILE", 
 "parent":"/2016", 
 "name":"Q1-Notes.docx", 
 "changeTime":1456867234262, 
 "size":3252, 
 "hash":"f8e25", 
 "access":"COLLABORATOR", 
 "etag":"ePM==" 
 }

Description of Response

Response body for move▾

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to move the entry.
  • 404 - If createParents is set to true, the given sourcePath does not exist. If createParents is set to false, either the given sourcePath does not exist or a parent of the destinationPath does not exist.
  • 409 - A conflict was encountered while trying to move the entry.
  • 413 - A limit on HCP Anywhere was encountered while trying to move the entry. The error message contained in the reponse body includes more information.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to move the entry. The request should be retried.

Search

Searches for files and folders in HCP Anywhere.

This is done by specifying a folder in which to search. This request also searches the subfolders of the folder being searched.

URI

/fss/public/path/search

HTTP Method

POST

Parameters

  • Request Body:
    Request body for search▾

    Request body parameters needed to search for files and folders. This request body needs to be JSON.

Example Request

Here is an example of a search request in cURL. In this example, the folder to search within is "Work-Projects", the string to search for is "Sales", and the access token is "abc=". In an actual request, the access token is much longer. 

 curl -kiX POST https://anywhere.example.com/fss/public/path/search -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d '{"path": "/Work-Projects", "substring": "Sales", "maxResults": "2"}'

Example Response

The response for a successful search request. In an actual response, the etags are much longer.

 "changeTime": 1456870069777, 
 "access": "COLLABORATOR", 
 "etag": "ytn==" 
 }, 
 { 
 "type": "FOLDER", 
 "parent": "/Work-Projects/Sales-Reports",
 "name": "Sales-Q1", 
 "changeTime": 1456870095676, 
 "access": "COLLABORATOR", 
 "etag": "jnd==" 
 } 
 ] 
 }

Description of Response

Response body for search▾

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Bad or expired access token.
  • 403 - Access to the resource was forbidden while trying to search.
