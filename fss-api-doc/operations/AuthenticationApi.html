Authentication

This page consists of the Authentication endpoints for the HCP Anywhere File Sync and Share API.

For each endpoint, the following information is provided (if applicable):

  • Description of the operation the endpoint performs

  • HTTP method for the endpoint

  • Example request

  • Response codes and description of what each response code means

Authenticate

Authenticates the user and returns an access token. The access token is then provided as a header in the other File Sync and Share API endpoints. The settings in the Management Console determine the amount of time the access token remains active.

The HCP Anywhere File Sync and Share API supports both basic authentication and form authentication.

URI

/fss/public/login/oauth

HTTP Method

POST

Parameters

  • Request Body:
    Request body for basic authentication▾

    Basic Authentication: Request body parameters needed to acquire an access token using basic authentication. This request body needs to be x-www-form-urlencoded.

  • Request Body:
    Request body for form authentication▾

    Form Authentication: Request body parameters needed to acquire an access token using form authentication. This request body needs to be JSON.

Example Request

Basic authentication:

Here is an example of a basic authentication request in cURL. In this example, the username is "jsmith" and the password is "mypassword". 

 curl https://anywhere.example.com/fss/public/login/oauth -iku "jsmith:mypassword" -H "Content-Type: application/x-www-form-urlencoded" -H "Accept: application/json"  -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -d 'grant_type=urn:hds:oauth:negotiate-client'

Form authentication:

Here is an example of a form authentication request in cURL. In this example, the username is "jsmith" and the password is "mypassword". 

 curl -kiX POST https://api.anywhere.example.com/fss/public/login/oauth -H "X-HCPAW-FSS-API-VERSION: 2.1.1" -H "Accept: application/json" -H "Content-Type: application/json" -d '{"username": "jsmith", "password": "mypassword", "grant_type": "urn:hds:oauth:negotiate-client"}'

Example Response

The response for a successful authentication request. The response is the same for basic and form authentication. In an actual response, the access token is much longer. 

 { 
 "access_token": "abc=", 
 "token_type":  "Bearer", 
 "expires_in": 2592000 
 }

Description of Response

Response body for authenticate▾

Response Codes

  • 200 - Operation successful.
  • 400 - Bad request. The error message contained in the reponse body includes more information.
  • 401 - Authentication failed.
  • 403 - Access to the resource was forbidden while trying to authenticate the user.
  • 500 - Internal server error.
  • 503 - The server was temporarily unable to authenticate the user. The request should be retried.
