List filesystems

This request lists a user's filesystems. For each user, these are all identified as separate filesystems:

•The user's private HCP Anywhere filesystem

•Each shared folder

• Each team folder

•Each backup parent folder

URI

/fss/public/user/filesystems/list

HTTP Method

POST

Parameters

There are no parameters for a user/filesystems/list request.

Example Request

This section contains an example of a user/filesystems/list request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/user/filesystems/list -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0"

Description of Response

This section describes the contents of a user/filesystems/list response.

Response content:

•filesystems (required): Object

A list of the user's filesystems.

opath (required): String

The path to the filesystem. For the user's private HCP Anywhere filesystem, the path is always "/".

ofsId (required): String

A unique identifying string assigned to the filesystem by HCP Anywhere.

otype (required): Enum

The type of filesystem. For the user's private HCP Anywhere filesystem, the type is always "PRIVATE".

oownerInfo (conditional): Object

Information about the owner of the filesystem. This information is only returned for the user's private HCP Anywhere filesystem and the shared folders in which the user is a member.

–username (required) : String

The user's username. The username is acquired from the authentication provider (SAML or AD) of that user.

–displayName (required) : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of that user.

–userId (required) : String

The unique identifying string assigned to the user by HCP Anywhere.

oteamFolder (conditional): Object

The additional settings that apply to team folders. This information is only returned for team folders.

–contact: String

The contact of the team folder.

–quota: Long

The total storage capacity of the team folder. This number is returned in bytes.
