List folder contents

This request lists the items (files and folders) in a folder.

URI

/fss/public/folder/entries/list

HTTP Method

POST

Parameters

The folder/entries/list endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the folder.

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of items you are.

•pageSize (optional) : Integer

The maximum number of items to return in a request. The default is 100.

•sortingCriterion (optional): Enum

The way in which the items are sorted in the response body (NAME, MODIFY_DATE, SIZE). If you enter "NAME", the items are sorted alphabetically from A to Z. If you enter "MODIFY_DATE", the items are sorted by the times at which they were modified from the most recently modified file to the least recently modified file. If you enter "SIZE", the items are sorted by size from largest to smallest.

The default is "NAME". The way in which the items are sorted also depends on the "sortingDirection" parameter.

•sortingDirection (optional): Enum

The direction in which the list is sorted. To ascend through the list of items, enter "ASC". To descend through the list of items, enter "DESC".

For example, if "NAME" is supplied for the "sortingCriterion" parameter and "ASC" is supplied for this parameter, the items are returned in alphabetical order from A to Z. If "DESC" is supplied, the items are sorted alphabetically from Z to A.

The default is "ASC".

•viewAtTime (optional): Long

The point in time at which to list the items. This needs to be supplied in Unix format in milliseconds. For example, if viewAtTime is set to "1480569120000" (12:00pm on December 1, 2016), the given files and folders are listed as they were at 12:00pm on December 1, 2016. To find the different times available to you, use a user/restorePoints/list request. A restore point is a snapshot of the filesystem at a certain point in time.

If this parameter is not included in the request, the items in the folder are listed as they are currently.

•IncludeDeleted (optional): Boolean

If true, any items that have been deleted in the folder and are still in the system are returned along with the existing items. The "state" of the item returned in the response body identifies whether the file is currently deleted or not.

Example Request

This section contains an example of a folder/entries/list request in cURL. In this example, the folder to list the contents of is "Sales-Reports" and the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/folder/entries/list -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"path":"/Work-Projects/Sales-Reports"}'

Description of Response

This section describes the contents of a folder/entries/list response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of folder items you are. You can include this page token in the body of a subsequent request to return the next set of folder items.

•entries (required) : Object

A list of the items in the folder. The number of items returned in the response is at most the number specified in the pageSize parameter.

otype (required) : Enum

The type of item (FILE or FOLDER).

oparent (required) : String

The path to the folder that contains the item.

oname (required) : String

The name of the file or folder.

ochangeTime (required) : Long

The time at which the item was last accessed or modified. This time is returned in Unix format.

osize (conditional) : Long

If the item is a file, the size of the file in bytes.

ohash (conditional) : String

If the item is a file, the hex encoded sha-384 hash of the file.

oaccess (required) : String

The type of operations the given user can perform on the item:

–NO ACCESS — The person cannot access the file or folder.

–VIEWER — The person can view and download the file or folder. However, if the item is a folder, the person cannot upload files to the folder.

–COLLABORATOR — The person can download, upload, move, and delete files and folders.

oetag (required) : String

The etag of the item. The etag identifies the version of the file or folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

ostate (required) : String

The state of the item (CREATE or DELETE).

osharing (conditional): Object

If the item is a shared folder, team folder, or mobilized data share or within one of these, some additional folder information.

–canUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

–isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized data shares are mount points, although the items within them are not.

–label (conditional) : String

If the item is a shared or team folder, the label of the folder.

–type:Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

–teamFolder (conditional): Object

If the item is a team folder or within a team folder, some additional team folder information:

•contact: String

The contact of the team folder.

•quota: Long

The total storage capacity of the team folder. This is returned in bytes.

obackup (conditional): Object

If the item is a backup file or folder, some additional backup information:

–isActive: Boolean

If true, the backup folder is connected to the HCP Anywhere desktop application. 

The backup folder can be disconnected from the desktop application if the desktop application is deregistered or if the backup feature is disabled by the system administrator.

–isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. The backup parent folder, which is automatically created when backup is enabled and the given user's desktop application is installed, is always a mount point.

–operatingSystem: Enum

The type of device on which the desktop application is installed (WINDOWS, WINDOWS NSC, or MAC). For version 3.0 or later of the Windows desktop application, the operatingSystem is "WINDOWS NSC".
