Restore deleted folder

This request restores a deleted folder in HCP Anywhere.

When a folder is restored, the contents of the folder are also restored.

URI

/fss/public/folder/restore

HTTP Method

POST

Parameters

The folder/restore endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the folder.

•deltaSeconds (required) : Integar

The point in time at which to restore the folder. This needs to be supplied in seconds. If deltaSeconds is set to "86400" (1 day), for example, the given folder is restored as it was one day prior to being deleted.

Example Request

This section contains an example of a folder/restore request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/folder/restore -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Previous Projects", "deltaSeconds": "864000"}'

Example Response

This section contains an example of a response to a successful folder/restore request.

{
    "type":"FOLDER",
    "parent":"/2015/",
    "name":"Previous Projects",
    "changeTime":1467830900704,
    "access":"COLLABORATOR",
    "etag":"xyz==",
    "state":"CREATE"
}

Description of Response

This section describes the contents of a folder/restore response.

Response content:

•type (required) : String

The type of item (FILE or FOLDER). For this response, the type is always FOLDER.

•parent (required) : String

The path to the folder that contains the folder.

•name (required) : String

The name of the folder.

•changeTime (required) : Long

The time at which the folder was last accessed or modified. This time is returned in Unix format.

•etag (required) : String

The etag of the folder. The etag identifies the version of the folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

•state (required) : Enum

The state of the folder (CREATE or DELETE).

•sharing (conditional): Object

If the folder is within a shared folder, team folder, or mobilized share, some additional folder information.

ocanUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

oisMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized shares are always mount points although the items within them are not.

olabel (conditional) : String

If the item is a shared or team folder, the label of the folder.

otype: Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

oteamFolder (conditional): Object

If the folder is created within a team folder, some additional team folder information:

–contact: String

The contact of the team folder.

–quota: Long

The total storage capacity of the team folder. This is returned in bytes.

•backup (conditional): Object

If the folder is a backup folder, some additional backup information:

oisActive: Boolean

If true, the backup folder is connected to the HCP Anywhere desktop application. 

The backup folder can be disconnected from the desktop application if the desktop application is deregistered or if the backup feature is disabled by the system administrator.

oisMountPoint: Boolean

If true, the folder is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. The backup parent folder, which is automatically created when the desktop application is installed with backup enabled, is always a mount point.

ooperatingSystem: Enum

The type of device on which the desktop application is installed (WINDOWS, WINDOWS NSC, or MAC). For version 3.0 or later of the Windows desktop application, the operatingSystem is "WINDOWS NSC".

ohasAssociatedClient: Boolean

Each backup file or folder is associated with the desktop application that performed its backup. If true, the desktop application associated with the backup folder is currently registered. If false, the desktop application associated with the backup folder is currently deregistered.

Note: An application can be deregistered if:

•You deregister it in the User Portal in the My Devices window

• Your administrator deregisters it in the HCP Anywhere Management Console
