Clear login credentials

This request clears the login credentials for an HCP Anywhere application.

For the HCP Anywhere desktop application, mobile application, and Outlook Add-in, you can clear the login credentials for each application individually. For the third-party applications registered to a user's accounts, you can only clear the login credentials for all of the third-party applications at once. However, since all the third-party applications registered to a user's account share a single access token, you cannot clear the login credentials for a specific third-party application.

URI

/fss/public/client/clearCredentials

HTTP Method

POST

Parameters

The client/clearCredentials endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•id (required) : String

The unique string assigned to the application by HCP Anywhere. 

You can acquire an application's id using a client/list request.

Example Request

This section contains an example of a client/clearCredentials request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/client/clearCredentials -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"id": "mndqlcngh"}'

Example Response

This section contains an example of a response to a successful client/clearCredentials request.

{
    "id": "mndqlcngh",
    "nickname": "Windows Computer",
    "os": "WINDOWS",
    "version": "2.1.1.2",
    "lastAccess": 1467921091077,
    "needsUpgrade": false,
    "inRecovery": false
}

Description of Response

This section describes the contents of a client/clearCredentials response.

Response content:

•id (required) : String

The unique string assigned to the application by HCP Anywhere.

•nickname (required) : String

The new name of the application.

•os (required) : String

The type of device that the HCP Anywhere application is installed on (for example, Windows or Mac). For the User Portal, the os is always "Browser".

•version (required) : Long

The HCP Anywhere version of the application. For the User Portal and any third-party applications, the version is always "None".

•lastAccess (required) : Long

The time at which the application was last accessed by the user. This time is returned in Unix format.

•needsUpgrade (required): Boolean

If true, an upgrade is available for the application. An upgrade becomes available to an application when the HCP Anywhere system is upgraded.

•inRecovery (required): Boolean

If true, the application is in the middle of the recovery process. An application goes into the recovery process the first time it connects to the HCP Anywhere system after the HCP Anywhere system has been recovered.
