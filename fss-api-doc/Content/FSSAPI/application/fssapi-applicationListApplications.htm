List applications

This request lists the applications registered to a user's HCP Anywhere account.

Most of the HCP Anywhere applications registered with a user's account are listed individually (for example, Windows application, Mac application, etc). However, the third-party applications registered to a user's HCP Anywhere account are listed as a single application with the name "FSS API". All third-party applications registered to a user's account share a single access token.

URI

/fss/public/client/list

HTTP Method

POST

Parameters

There are no parameters for a client/list request.

Example Request

This section contains an example of a client/list request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/client/list -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0"

Description of Response

This section describes the contents of a client/list response.

Response content:

•clients (required): Object

The applications registered with the user's HCP Anywhere account:

oid (required) : String

A unique string assigned to the application by HCP Anywhere.

onickname (required) : String

The name of the application. For the User Portal, the name of the application is always "User Portal". For third-party applications, the name of the application is always "FSS API".

For the other HCP Anywhere applications, the name of the application is, by default, the actual name of the device (for example, USWAL-JSMITH). However, you can update the names of these other HCP Anywhere applications using a client/rename request.

oos (required) : String

The type of device that the HCP Anywhere application is installed on (for example, Windows or Mac). For the User Portal, the os is always "Browser".

oversion (required) : String

The HCP Anywhere version of the application. For the User Portal and any third-party applications, the version is always "None".

olastAccess (required) : Long

The last time the application was accessed by the user. This time is returned in Unix format.

oneedsUpgrade (required): Boolean

If true, an upgrade is available for the application. An upgrade becomes available to an application when the HCP Anywhere system is upgraded.

oinRecovery (required): Boolean

If true, the application is in the middle of the recovery process. An application goes into the recovery process the first time it connects to the HCP Anywhere system after the HCP Anywhere system has been recovered.
