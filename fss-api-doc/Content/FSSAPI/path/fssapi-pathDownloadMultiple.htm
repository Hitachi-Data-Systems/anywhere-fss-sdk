Download multiple items

This request downloads multiple files and folders in HCP Anywhere. This endpoint can be used to:

•Download files and folders as they currently are.

•Download files and folders as they were at a previous point in time.

With this endpoint, you can download a single item or multiple items. The downloaded items are returned in a .zip file.

URI

/fss/public/path/zip

HTTP Method

POST

Parameters

The path/zip endpoint supports supplying the path/zip parameters in a request body or as query parameters. When using a request body, the request body must be in JSON format.

Request body or query parameters:

•pathList (required): Array

The path to each item you want to download.

•pointInTime (optional): Long

The point in time at which to download the items. This needs to be supplied in Unix format in milliseconds. For example, if pointInTime is set to "1480569120000" (12:00pm on December 1, 2016), the request returns the files and folders as they were at 12:00pm on December 1, 2016. To find the different times available to you, use a user/restorePoints/list request. A restore point is a snapshot of the filesystem at a certain point in time.

If this parameter is not included in the request, the files and folders in the pathList are downloaded as they are currently.

Example Request

This section contains an example of a JSON and form path/zip request in cURL. In these two examples, the authentication token is "abc=". In an actual request, the authentication token is much longer.

The "-o" option is supplied in each request. The "-o" option allows you to save the file to a local path on your computer instead of having the file streamed to your terminal in bytes. If you are using the "-o" option, do not include an "i" in the request (such as "kiX"). This corrupts the file.

JSON based request:

curl -kX POST https://anywhere.example.com/fss/public/path/zip -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/zip" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"pathList": ["/Q1-Overview.docx", "/Q2-Overview.docx"]}' -o "C:\JSMITH\Desktop\Overviews.zip"


Form based request:

curl -kX POST "https://anywhere.example.com/fss/public/path/zip?pathList=/Q1-Overview.docx&pathList=Q2-Overview.docx" -H "Authorization: Bearer abc=" -H "Content-Type: application/x-www-form-urlencoded" -H "Accept: application/zip" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -o "C:\JSMITH\Desktop\Overviews.zip"

Example Response

In response to a successful path/zip response, no information is returned in the response body. Instead, the .zip file is downloaded to the location that you specified in the "-o" option of the request.
