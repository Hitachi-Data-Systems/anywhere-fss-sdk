Move / rename

This request moves, renames, or both moves and renames a file or folder in HCP Anywhere.

URI

/fss/public/path/move

HTTP method

POST

Parameters

The path/move endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•sourcePath (required) : String

The current path of the file or folder that you want to move.

•destinationPath (required) : String

The path where the file or folder will be moved. If a different name is entered for the last item in the path, the file or folder is renamed.

•etag (required) : String

The etag of the file or folder. The etag identifies the version of the file or folder and is part of the file or folder metadata.

•createParents (optional) : Boolean

If true, the request creates any folders that do not exist in the path you specified in the destinationPath paramter. If false, the request fails if any folders in the path do not exist. The default is false.

Example request

This section contains an example of a path/move request in cURL. In this example, the file to move is "Q1-Notes.docx" and the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/path/move -H "Accept: application/json" -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"sourcePath": "/Q1-Notes.docx", "destinationPath": "/2016/Q1-Notes.docx", "etag": "ePM==", "createParents": true}'

Example response

This section contains an example of a response to a successful path/move request. In an actual response, the authentication token and hash are much longer.

{
    "type":"FILE",
    "parent":"/2016",
    "name":"Q1-Notes.docx",
    "changeTime":1456867234262, 
    "size":3252,
    "hash":"f8e25",
    "access":"COLLABORATOR",
    "etag":"ePM==",
    "state": "CREATE"
}

Description of response

This section describes the contents of a path/move response.

Response content:

•type (required) : Enum

The type of item (FILE or FOLDER).

•parent (required) : String

The path to the folder that contains the item.

•name (required) : String

The name of the item.

•changeTime (required) : Long

The time at which the item was last accessed or modified. This time is returned in Unix format.

•size (conditional) : Long

If the item is a file, the size of the file in bytes.

•hash (conditional) : String

If the item is a file, the hex encoded sha-384 hash of the file.

•access (required) : Enum

The type of operations the given user can perform on the item:

oNO ACCESS — The user cannot access the file or folder.

oVIEWER — The user can view and download the file or folder. However, if the entry is a folder, the user cannot upload files to the folder.

oCOLLABORATOR — The user can download, upload, move, and delete files and folders.

•etag (required) : String

The etag of the item. The etag identifies the version of the file or folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

•state (required) : Enum

The state of the file (CREATE or DELETE).

•sharing (conditional): Object

If the item is a shared folder, team folder, or mobilized share or within one of these, some additional folder information.

ocanUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

oisMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized shares are always mount points although the items within them are not.

olabel (conditional) : String

If the item is a shared or team folder, the label of the folder.

otype: Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

oteamFolder (conditional): Object

If the item is a team folder, some additional team folder information:

–contact: String

The contact of the team folder.

–quota: Long

The total storage capacity of the team folder. This is returned in bytes.

•backup (conditional): Object

If the item is a backup file or folder, some additional backup information:

oisActive: Boolean

If true, the backup folder is connected to the HCP Anywhere desktop application. 

The backup folder can be disconnected from the desktop application if the desktop application is deregistered or if the backup feature is disabled by the system administrator.

oisMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. The backup parent folder, which is automatically created when the desktop application is installed with backup enabled, is always a mount point.

ooperatingSystem: Enum

The type of device on which the desktop application is installed (WINDOWS, WINDOWS NSC, or MAC). For version 3.0 or later of the Windows desktop application, the operatingSystem is "WINDOWS NSC".

ohasAssociatedClient: Boolean

Each backup file or folder is associated with the desktop application that performed its backup. If true, the desktop application associated with the backup folder is currently registered. If false, the desktop application associated with the backup folder is currently deregistered.

Note: An application can be deregistered if:

•You deregister it in the User Portal in the My Devices window

• Your administrator deregisters it in the HCP Anywhere Management Console
