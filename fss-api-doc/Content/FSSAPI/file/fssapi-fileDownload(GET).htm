Download file (GET)

This request downloads a file.

Typically, you use a GET request when making a request from a web browser

URI

/fss/public/file/stream/read

HTTP Method

GET

Parameters

The file/stream/read (GET) endpoint requires only query parameters, but can also contain header parameters.

Query parameters:

•path (required) : String

The path of the file you want to download.

•forceDownload (optional): Boolean

If an etag is included in the request and forceDownload is set to true, the version of the file that relates to the etag is returned, even if the etag identifies an older version of the file. The default is false.

Header parameters:

•etag (optional) : String

The etag of the file you want to download. The etag identifies the version of the file and is part of the file metadata.

If the etag does not match the file specified in the path, the file specified in the path is returned instead. Possible reasons for an etag that does not match include an etag is simply incorrect, an etag that belongs to another file, an etag that belongs to an older version of the file you want to download.

If the etag matches the file specified in the path, the file is not returned because the local file is already up-to-date. Even if the etag matches the file, you can still download the file using the "forceDownload" parameter.

If the etag is not included in the request body, the latest version of the file is downloaded.

Example Request

This section contains an example of a file/stream/read (GET) request in cURL. In this example, the file being downloaded is "example.docx", the outdated etag is "xyz==", and the authentication token is "abc=". In an actual request, the etag and authentication token are much longer.

curl -kiX GET "https://anywhere.example.com/fss/public/file/stream/read?path=/Work-Projects/example.docx" -H "Accept: application/octet-stream" -H "Content-Type: application/json" -H "If-None-Match: xyz==" -H "Authorization: Bearer abc=" -H "X-HCPAW-FSS-API-VERSION: 3.0.0"

Description of Response

The response body for a link/stream/read (GET) contains the contents of the file that is downloaded. The contents of the file are streamed to the terminal.

If you do not want the contents of the file to be streamed to the terminal, you can include the "-o" option in your request. The "-o" option allows you to specify a local path on your computer where the file is saved.

If you are using the "-o" option, do not include an "i" in the request (such as "kiX"). This corrupts the file.
