List read access

This request lists the times at which a file in HCP Anywhere was accessed, but not through a link.

URI

/fss/public/file/readAccess/list

HTTP Method

POST

Parameters

The file/readAccess/list endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the file.

•pageSize (optional) : Integer

The maximum number of items to return in the request. The default is 100.

•pageAction (optional): Enum

The portion of the access events to return:

o FIRST — Returns the newest access events. For example, if your pageSize is 3, then the newest three access events are listed.

oLAST — Returns the oldest available access events. For example, if your pageSize is 3, then the oldest three events are listed.

oNEXT — Returns the next set of access events based on your position in the list of events. Your position in the list of access events is determined by the page token. A page token is returned in response to a successful request.

oPREV — Returns the previous set of access events based on your position in the events. Your position in the list of access events is determined by the page token. A page token is returned in response to a successful request.

Note: If a pageAction and pageToken are not included in the request, the newest set of access events are returned.

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of access events you are.

•showPrivate (optional): Boolean

The showPrivate parameter only applies to files within a shared or team folder. If true, only access events that occurred while the user was a member of the shared folder are returned. If false, all available access events for the file (including times the file was accessed before the user joined to the folder) are returned. The default is true.

Example Request

This section contains an example of a file/readAccess/list request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/file/readAccess/list -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"path": "/Q1 Review.docx", "pageSize": 1, "pageAction": "FIRST"}'

Example Response

This section contains an example of a response to a successful file/readAccess/list request.

{
    "pageToken": "MnRQq",
    "entries": [
    {
        "fileName": "Q1 review.docx",
        "username": "Jim Smith",
        "size": 12934,
        "timestamp": 1477922158457
       }
   ]
}

Description of Response

This section describes the contents of a file/readAccess/list response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of access events you are. You can supply this page token in the body of a request to return the next set of access events.

•entries (required): Object

The list of access events for the file:

ofileName (required): String

The name of the file that was accessed.

ousername (required): String

The username of the person who accessed the link. 

osize (required) : Long

The size of the file in bytes.

otimestamp (required): Long

The time at which the event occurred. This time is returned in Unix format.
