List versions of a file

This request lists the versions of a file.

A file version is created each time a file is modified (for example, when someone edits a file). HCP Anywhere keeps each file version for a limited amount of time and keeps a limited number of versions. The number of versions that are kept depends on your organization's HCP Anywhere settings.

Tip:  You can restore an older version of a file using a file/version/promote request.

URI

/fss/public/file/version/list

HTTP Method

POST

Parameters

The file/version/list endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the file.

•pageSize (optional) : Integer

The maximum number of items to return in the request. The default is 100.

•pageAction (optional): Enum

The portion of the versions to return:

o FIRST — Returns the newest versions of the file. For example, if your pageSize is 3, then the newest three versions of the file are listed.

oLAST — Returns the oldest available versions of the file. For example, if your pageSize is 3, then the oldest three versions of the file are listed.

oNEXT — Returns the next versions of the file based on your position in the list of versions. Your position in the list of versions is determined by the page token. A page token is returned in response to a successful request.

oPREV — Returns the previous versions of the file based on your position in the list of versions. Your position in the list of versions is determined by the page token. A page token is returned in response to a successful request.

Note: If a pageAction and pageToken are not included in the request, the newest versions of the file are returned.

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of versions you are.

•showPrivate (optional): Boolean

The showPrivate parameter only applies to files within a shared or team folder. If true, only versions of file that were created while the user was a member of the shared folder are returned. If false, all available versions for the file (including versions created before the user joined to the folder) are returned. The default is true.

Example Request

This section contains an example of a file/version/listrequest in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/file/version/list -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Q1 Overview.docx","pageSize": 2, "pageAction": "First"}'

Description of Response

This section describes the contents of a file/version/list response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of versions you are. You can supply this page token in the body of a request to return the next set of versions.

•entries (required): Object

The versions of the file:

oevent (required): String

The event that caused the version to be created. For example, event.label.update means the file was modified.

ousername (required) : String

The user's username. The username is acquired from the authentication provider (SAML or AD) of that user.

osize (required) : Long

The size of the file in bytes.

otimestamp (required): Long

The time at which the version was created. This time is returned in Unix format.

ocurrentVersion (required): Boolean

If true, the version listed is the current version of the file. If false, the version listed is not the current version of the file.

oetag (required) : String

The etag of the file. The etag identifies the version of the file and is supplied as a parameter in some of the other File Sync and Share API endpoints.
