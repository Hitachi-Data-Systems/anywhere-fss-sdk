List file conflicts

This request lists the existing conflict files in a user's HCP Anywhere account.

HCP Anywhere creates a conflict file when two people edit a file at the same time. Typically, conflicts occur in collaborative folders since multiple people have access to the same files.

URI

/fss/public/file/conflict/list

HTTP Method

POST

Parameters

There are no parameters for a file/conflict/list request.

Example Request

In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/file/conflict/list -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0"

Example Response

This section contains an example of a response to a successful file/conflict/list request.

{
    "entries": [
      {
          "type": "FILE",
          "parent": "/",
          "name": "Q1 Overview ( jsmith's conflict 2016-10-19 ).docx",
          "changeTime": 1469740333000,
          "size": 13529,
          "hash": "91a94",
          "access": "COLLABORATOR",
          "etag": "YtRQq==",
          "state": "CREATE"
      }
   ]
}

Description of Response

This section describes the contents of a file/conflict/list response.

Response content:

•entries (required): Object

A list of conflict files in the user's HCP Anywhere:

otype (required) : Enum

The type of item (FILE or FOLDER). For this response, the type is always FILE.

oparent (required) : String

The path to the folder that contains the file.

oname (required) : String

The name of the file.

ochangeTime (required) : Long

The time at which the file was last accessed or modified. This time is returned in Unix format.

osize (required) : Long

The size of the file in bytes.

ohash (required) : String

The hex encoded SHA-384 hash of the file.

oaccess (required) : String

The type of operations the given user can perform on thefile:

–NO ACCESS — The person cannot access the file.

–VIEWER — The person can view and download the file.

–COLLABORATOR — The person can download, upload, move, and delete the file.

oetag (required) : String

The etag of the file. The etag identifies the version of the file and is supplied as a parameter in some of the other File Sync and Share API endpoints.

ostate (required) : String

The state of the item (CREATE or DELETE).

osharing (conditional): Object

If the file is within a shared folder, team folder, or mobilized share, some additional folder information.

–canUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

–isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized shares are always mount points although the items within them are not.

–label (conditional) : String

If the item is a shared or team folder, the label of the folder.

–type:Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

–teamFolder (conditional): Object

If the item is a team folder or within a team folder, some additional team folder information:

•contact: String

The contact of the team folder.

•quota: Long

The total storage capacity of the team folder. This is returned in bytes.
