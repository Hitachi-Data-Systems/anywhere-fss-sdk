Modify account settings

This request modifies a user's account settings. A user's account settings include the times when the user receives email notifications and the display language for the HCP Anywhere User Portal.

URI

/fss/public/user/settings/update

HTTP Method

POST

Parameters

The user/settings/update endpoint requires only request body parameters. The request body needs to be in JSON format.

Important:  

•Each parameter for a user/settings/update request is optional. However, at least one user/settings/update parameter needs to be included in the request body to successfully make a request.

•In order to receive email notifications from HCP Anywhere, your system administrator must have email notifications enabled on the HCP Anywhere server.

If you are an HCP Anywhere administrator, see the HCP Anywhere Administrator Help for instructions on enabling email notifications.

Request body parameters:

•email (optional): String

The user's email address. You can only add an email address if one has not already been set for the user.

•emailQuota (optional): Boolean

If true, the user is sent an email when their HCP Anywhere storage is nearing capacity.

•emailDevices (optional): Boolean

If true, the user is sent an email when a device is registered to your account.

•emailAuth (optional): Boolean

If true, the user is sent an email when their is a login failure for their account.

•emailShareFile (optional): Boolean

If true, the user is sent an email when one of your files or folders are accessed through a link.

•emailUpload (optional): boolean

If true, the user is sent an email when someone uploads a file to a folder that is accessed through a link that the user shared.

•emailShareFolder (optional): Boolean

If true, the user is sent an email when they are invited to a shared or team folder.

•language (optional): Enum

The display language for the user's HCP Anywhere User Portal. You can enter:

oen-US — English

ozh-CN — Simplified Chinese

ofr-CA — French Canadian

oja — Japanese

oko — Korean

oes — Spanish (Latin America)

ode — German

Example Request

This section contains an example of an user/settings/update request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/user/settings/update -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"emailQuota": "true"}'

Description of Response

This section describes the contents of an user/settings/update response.

Response content:

•username (required) : String

The user's username. The username is acquired from the authentication provider (SAML or AD) of that user.

•displayName (required) : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of that user.

•userid (required): String

A unique identifying string assigned to the user by HCP Anywhere.

•uniqueId (required) : String

A unique identifying string assigned to the user by the user's authentication provider (AD or SAML). For AD, this is the user's Security Identifier (SID).

•quotaBytes (required): Long

The total amount of HCP Anywhere storage space allotted to the user. This number is returned in bytes. The total amount of HCP Anywhere storage space allotted to a user is determined by the profiles which they are in.

•usageBytes (required): Long

The amount of storage space the user is currently using in HCP Anywhere. This number is returned in bytes.

•hasAuthProviderEmail (required): Boolean

If true, the user has an email address configured by their authentication provider (AD or SAML).

•email (required): Long

The user's email address. If the user has email notifications configured, this email address receives those notifications.

•linkSettings (required) : Object

The user's default link settings. The link settings are configured by the HCP Anywhere administrator.

oreadEnabled or sharingEnabled (required) : Boolean

If true, link sharing is enabled on the HCP Anywhere server.

In release 3.0, "sharingEnabled" was renamed "readEnabled". If an application makes a request using the 2.1.1 File Sync and Share API, "sharingEnabled" is still returned even when making a request to a 3.0 or later HCP Anywhere server. This ensures backwards compatibility.

opublicReadEnabled or publicSharingEnabled (required) : Boolean

If true, public link sharing is enabled on the HCP Anywhere server.

In release 3.0, "publicSharingEnabled" was renamed "publicReadEnabled". If an application is making requests using the 2.1.1 File Sync and Share API, "publicSharingEnabled" is still returned even when making a request to a 3.0 or later HCP Anywhere server. This ensures backwards compatibility. This ensures backwards compatibility.

ouploadEnabled (required) : Boolean

If true, upload link sharing is enabled on the HCP Anywhere server.

opublicUploadEnabled (required) : Boolean

If true, public upload link sharing is enabled on the HCP Anywhere server.

osharingDefaultPublic (required) : Boolean

If true, public links are the default way for the user to share. If false, internal links are the default way for the user to share.

odefaultDaysToShare (required) : Integer

The default number of days a link created by the user remains active.

omaxDaysToShare (required) : Integer

The maximum number of days a link created by the user can remain active.

•accountSettings (required): String

The user's account settings.

oemailQuota (required): Boolean

If true, the user is sent an email when their HCP Anywhere account is nearing storage capacity.

oemailDevices (required): Boolean

If true, the user is sent an email when a device is registered to their account.

oemailAuth (required): Boolean

If true, the user is sent an email when their is a login failure occurs for their account.

oemailShareFile (required): Boolean

If true, the user is sent an email when one of their files or folders are accessed through a link.

oemailUpload (required): boolean

If true, the user is sent an email when someone uploads a file to a folder that was accessed through one of the user's links.

oemailShareFolder (required): Boolean

If true, the user is sent an email when they are invited to a shared or team folder.

oallowedClients (required): Array

The type of HCP Anywhere applications that the person can use (install and register). If "PUBLIC_API" is returned, the user can use the File Sync and Share API. If "Windows_NSC" or "MacOSX_NSC" are returned, the user can use the 3.0 version of the desktop applications. If "Windows" or "Mac" are returned, the user can use desktop applications released before 3.0.

olanguage (required): Enum

The language settings for the user's HCP Anywhere account. The language settings are used to determine the language displayed in the HCP Anywhere User Portal

•teamFolderSettings (required): Object

The team folder settings for the user. A team folder is a type of collaborative folder that uses storage space set aside by the HCP Anywhere administrator instead of using a person's personal HCP Anywhere storage space.

oenabled (required): Boolean

If true, the user can create team folders.

oteamFolderQuotaThreshold (required): Long

The maximum quota that the user can set the team folder to have. This number is returned in bytes.

•fileCount (required): Integar

The number of files in the user's HCP Anywhere account.

•directoryCount (required): Integar

The number of folders in the user's HCP Anywhere account.

•conflicts (required): Integar

The number of conflict files in the user's HCP Anywhere account. A conflict occurs when two users modify the content of a file at the same time. Typically, conflicts occur in collaborative folders since multiple people have access to the same files.

•sharingEnabled (required): Boolean

If true, the user can create internal links and share folders. If false, the user cannot create internal links or share folders.

•registered (required): Boolean

If true, the user is registered with HCP Anywhere. A user becomes registered when they first access the HCP Anywhere User Portal or one of the HCP Anywhere applications. If false, the user is not registered with HCP Anywhere.

•configSettings (required): Object

Some additional user settings which are configured on the HCP Anywhere server.

omaxFileUploadSize (required): Long

The maximum size that a file can be when uploaded to HCP Anywhere. Files over this size cannot be uploaded. This size is returned in bytes.

oiOSAppURL (required): String

The URL where the HCP Anywhere iOS mobile application is located. From this location, the app can be downloaded.

oandroidAppURL (required): String

The URL where the HCP Anywhere android mobile application is located. From this location, the app can be downloaded.

owindowsPhoneAppURL (required): String

The URL where the HCP Anywhere Windows Pone mobile application is located. From this location, the app can be downloaded.

oenableiOSDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere iOS mobile application is available on the HCP Anywhere website. This link is located in the App Downloads window.

oenableAndroidDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere android mobile application is available on the HCP Anywhere website. This link is located in the App Downloads window.

oenableWindowsPhoneDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere Windows Phone mobile application is available on the HCP Anywhere website. This link is located in the App Downloads window.

oenableWindowsDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere Windows desktop application is available on the HCP Anywhere website. This link is located in the App Downloads window.

oenableMacDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere Mac desktop application is available on the HCP Anywhere website. This link is located in the App Downloads window.

oenableOutlookDownloadLink (required): Boolean

If true, the download link for the HCP Anywhere Outlook Add-in is available on the HCP Anywhere website. This link is located in the App Downloads window.

•inUpgrade (required): Boolean

If true, the HCP Anywhere system for the user is in the middle of an upgrade.

•providerId (required) : String

A unique identifying string assigned to the user's authentication provider (AD or SAML) by HCP Anywhere.
