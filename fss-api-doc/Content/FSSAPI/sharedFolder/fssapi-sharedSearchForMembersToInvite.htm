Search for people to invite

This request searches for people and groups who are part of the authentication networks (Active Directory and SAML) that are joined to HCP Anywhere.  It is these people and groups who can be invited to a collaborative folder.

Notes:  

•People and groups can be invited to a collaborative folder using a share/invite request.

•You can view the list of members in an Active Directory group by using a share/listGroupMembers request.

URI

/fss/public/share/invite/search

HTTP Method

POST

Parameters

The share/invite/search endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required): String

The path to the shared or team folder.

•filter (required): String

The string to search for in the HCP Anywhere network. A successful request returns people and groups in the network whose username or display name match some or all of string supplied in this parameter.

Example Request

This section contains an example of a share/invite/search request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/invite/search -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/share", "filter": "John"}'

Description of Response

This section describes the contents of a share/invite/search response.

Response content:

•users (required) : Object

The list of people and groups who match the search criteria:

odisplayName (required) : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of that user.

oname (required) : String

The user's username. The username is acquired from the identity provider (SAML or AD) of that user.

ouniqueId (required) : String

A unique identifying string assigned to the user by the user's authentication provider (AD or SAML). For AD, this is the user's Security Identifier (SID).

oproviderId (required) : String

The unique identifying string assigned to the user's authentication provider (AD or SAML) by HCP Anywhere.

orole (required) : Enum

The folder role of the member (VIEWER, COLLABORATOR, or MANAGER). The role identifies the operations a user can perform in the folder.

For a person who has never been invited to the folder, the person's role is "VIEWER".

ostate (required) : String

The user's state within the folder (INVITED, MEMBER, FORMER_MEMBER).

For a person who has never been invited to the folder, the person's state is "FORMER_MEMBER".

otype (required): Enum

Identifies whether the returned object is an individual person or a group (USER or GROUP).

oindividual (required): Boolean

If true, the folder member was invited individually to the folder. If false, the person was invited to the folder as part of a group.

•domain (required): String

The Active Directory domain that contains the user's user account.
