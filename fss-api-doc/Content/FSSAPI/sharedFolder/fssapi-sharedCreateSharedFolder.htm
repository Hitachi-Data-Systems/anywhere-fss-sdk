Create shared folder

This request converts a private folder into a shared folder.

When you share a folder with other people, the folder appears in their HCP Anywhere just as in your own. Changes that occur in the folder occur for all the folder members. For example, if you edit a document in the folder, those edits also occur for all the other members of the folder. Shared folders use the HCP Anywhere storage space of the person who creates the folder.

You can invite other people to join the shared folder using a share/invite request.

URI

/fss/public/share/create

HTTP Method

POST

Parameters

The share/create endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the private folder that you want to convert into a shared folder.

•label (required) : String

The label of the shared folder. Once the shared folder is created, the label cannot be changed. Whenever a new member is invited to the folder, the name of the folder is the folder label by default.

Example Request

This section contains an example of a share/create request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/create -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Marketing Project", "label": "Shared Folder"}'

Example Response

This section contains an example of a response to a successful share/create request.

{
    "path":"/Marketing Projects",
    "type":"SHARED",
    "label": "Shared Folder",
    "sync": true,
    "role": "Manager";
    "size":13047,
    "owner" {
        "username": "jsmith",
        "displayName": "John Smith",

   }
}

Description of Response

This section describes the contents of a share/create response.

Response content:

•path (required) : String

The path to the shared or team folder.

•type (required) : Enum

The type of shared folder (SHARED or TEAM). For this response, the type is always SHARED.

•label (required) : String

The label of the shared folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•owner (required): Object

Information about the Owner of the folder. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

ousername: String

The user's username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

odisplayName: String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ouserId: String

The unique identifying string assigned to the user by HCP Anywhere.
