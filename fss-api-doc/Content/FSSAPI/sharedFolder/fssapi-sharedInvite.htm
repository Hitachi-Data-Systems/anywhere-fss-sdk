Invite to folder

This request invites a person or group to a shared or team folder.

To invite a person or group to a folder, that person needs to be part of the Active Directory or SAML network connected to HCP Anywhere. You can search for people and groups in the Active Directory or SAML network using a share/invite/search request.

URI

/fss/public/share/invite

HTTP Method

POST

Parameters

The share/invite endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required): String

The path to the shared or team folder.

•uniqueId (required): String

The unique identifying string assigned to the user by the user's authentication provider (AD or SAML). You can acquire a user's uniqueId using a share/invite/search request.

•providerId (required): String

The unique identifying string assigned to the user's authentication provider (AD or SAML) by HCP Anywhere. You can acquire a user's providerId using a share/invite/search request.

•role (required): Enum

The role of the user who is being invited to the folder:

oViewer — Viewer is a read-only role. Viewers can view and download the files in the folder but cannot edit or add files to the folder.

oCollaborator — Collaborator is a read-write role. Collaborators can view, edit, and add files to the folder.

oManager — Manager is a read, write, and manage role. Managers can invite people to the folder, set member roles, adjust the folder storage capacity, change the folder contact information, and delete the folder for all the members. Managers can also view, edit, and add files to the folder, just like a Collaborator.

Important: You can only set the role to Manager when inviting someone to a team folder (not a shared folder). If you set the role to Manager when inviting someone to a shared folder, a 400 (Bad Request) code is returned in response to the request.

Example Request

This section contains an example of a share/invite request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/invite -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/Sales Team", "uniqueId": "S-1-5-21-3051028520-174195653-2777353501-2108", "providerId": "2305843009213693954", "role": "COLLABORATOR"}'

Example Response

This section contains an example of response to a successful share/invite request.

{
    "path":"/Sales Team",
    "type":"SHARED",
    "label": "Shared Folder",
    "sync": true,
    "role": "MANAGER",
    "size":13047,
    "owner" {
        "username": "jsmith",

        "userId": "8388608"
   }
}

Description of Response

This section describes the contents of a share/invite response.

Response content:

•path (required) : String

The path to the shared or team folder.

•type (required)

The type of shared folder (SHARED or TEAM).

•label (required) : String

The label of the shared or team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•owner (conditional) : String

If the folder is a shared folder, information about the Owner of the folder is returned. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

ousername: String

The user's username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

odisplayName: String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ouserId: String

The unique identifying string assigned to the user by HCP Anywhere.

•teamFolder (conditional): Object

If the folder is a team folder, some additional team folder settings:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
