Unshare folder

This request unshares an existing shared folder with all the folder members.

You can only unshare a folder if you are the Owner of the folder. The person who creates the shared folder is the Owner of the folder.

When you unshare a folder, the folder is removed from the HCP Anywhere accounts of all the other folder members but remains in your HCP Anywhere account as a private folder.

URI

/fss/public/share/unshare

HTTP Method

POST

Parameters

The share/unshare endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•path (required) : String

The path to the folder to unshare.

Example Request

This section contains an example of a share/unshare request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/unshare -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/Group Project"}'

Example Response

This section contains an example of a response to a successful share/unshare request.

{
    "type":"FOLDER",
    "parent":"/",
    "name":"Group Project",
    "changeTime":1468529039208,
    "access":"COLLABORATOR",
    "etag":"xyz==",
    "state":"CREATE"
}

Description of Response

This section describes the contents of a share/unshare response.

Response content:

•type (required) : Enum

The type of shared folder (SHARED or TEAM). For this response, the type is always SHARED.

•parent (required) : String

The path of the directory that contains the folder.

•name (required) : String

The name of the folder that was unshared.

•changeTime (required) : Long

The time at which the folder was last modified. This time is returned in Unix format. For this response, the time will always be when the folder was unshared.

•access (required) : Enum

The type of operations the given user can perform on the item:

oNO ACCESS — The user cannot access the file or folder.

oVIEWER — The user can view and download the file or folder. However, if the entry is a folder, the user cannot upload files to the folder.

oCOLLABORATOR — The user can download, upload, move, and delete files and folders.

For this response, the access is always COLLABORATOR.

•etag (required) : String

The etag of the folder. The etag identifies the version of the folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

•state (required) : String

The state of the item (Create or Delete). For this response, the state of the item will always be create since a private folder is created in place of the shared folder.
