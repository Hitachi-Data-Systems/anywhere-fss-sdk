Preserve group membership

This request preserves the groups in a shared or team folder as they are at the time of the request.

For each group in the folder, HCP Anywhere preserves the membership of the group by adding the members of the group to the folder as individuals.

URI

/fss/public/share/preserveMembership

HTTP Method

POST

Parameters

The share/preserveMembership endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•path (required): String

The path to the shared or team folder.

•removeGroups (required): Boolean

If true, the members of the groups in the shared or team folder are added to the folder as individuals and all the groups are deleted from the folder.

If false, the members of the groups in the shared or team folder are added to the folder as individuals and the groups continue to be tracked in Active Directory.

When a group is tracked, the group is updated at the end of each day to reflect its current membership in Active Directory. If a person has been added to the group in Active Directory, for example, that person is also invited to join the folder. Similarly, if a person has been removed from the group in Active Directory, that person is also removed from the folder.

Example Request

This section contains an example of a share/preserveMembership request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/preserveMembership -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Group Project", "removeGroups": true}'

Description of Response

This section describes the contents of a share/preserveMembership response.

Response content:

•members (required) : Object

Information about each member in the group:

oname (required) : String

The user's username. The username is acquired from the identity provider (SAML or AD) of that user.

ouniqueId (required) : String

The unique identifying string assigned to the user by the user's authentication provider (AD or SAML). For AD, this is the user's Security Identifier (SID).

oproviderId (required) : String

The unique identifying string assigned to the user's authentication provider (AD or SAML) by HCP Anywhere.

orole (required) : Enum

The folder role of the member (VIEWER, COLLABORATOR, or MANAGER). The role identifies the operations a user can perform in the folder.

ostate (required) : String

The user's state within the folder (INVITED, MEMBER, FORMER_MEMBER).

odomain (required): String

The Active Directory domain that contains the user's user account.

oaccountType (required): Enum

Identifies whether the returned object is an individual person or a group (USER or GROUP).

ouserInfo (required): Object

This object lists some additional information about the user:

–individual : Boolean

If true, the folder member was invited individually to the folder. If false, the person was invited to the folder as part of a group.

–displayName : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.
