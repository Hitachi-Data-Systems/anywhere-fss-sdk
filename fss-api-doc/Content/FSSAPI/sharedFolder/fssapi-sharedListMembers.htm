List members and groups in folder

This request lists the members and groups (current and former) in a shared or team folder.

URI

/fss/public/share/members

HTTP Method

POST

Parameters

The share/members endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the shared or team folder.

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of members you are.

•pageSize (optional) : Integer

The maximum number of items to return in a request. The default is 100.

•sortingCriterion (optional): Enum

The way in which the items are sorted in the response body (NAME, ROLE, STATE). If you enter "NAME", the items are sorted alphabetically from A to Z. If you enter "ROLE", the items are sorted by role (Viewer, Collaborator, and Manager). If you enter "STATE" , the items are sorted by status (Invited, Member).

The default is "NAME". The way in which the items are sorted also depends on the "sortingDirection" parameter.

•sortingDirection (optional): Enum

The direction in which the list is sorted. To ascend through the list of items, enter "ASC". To descend through the list of items, enter "DESC".

For example, if "NAME" is supplied for the "sortingCriterion" parameter and "ASC" is supplied for this parameter, the items are returned in alphabetical order from A to Z. If "DESC" is supplied, the items are sorted alphabetically from Z to A.

The default is "ASC".

Example Request

This section contains an example of a share/members request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/members -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/Marketing Project"}'

Description of Response

This section describes the contents of a share/members response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of folder members you are. You can supply this page token in the body of a request to return the next set of folder members.

•members (required) : Object

A list of members in the shared or team folder:

oname (required) : String

The user or group username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

ouniqueId (required) : String

The unique identifying string assigned to the user or group by their authentication provider (AD or SAML). For AD, this is the user's Security Identifier (SID).

oproviderId (required) : String

The unique identifying string assigned to the user's or group's authentication provider (AD or SAML) by HCP Anywhere.

orole (required) : Enum

The folder role of the member or group (VIEWER, COLLABORATOR, or MANAGER). The role identifies the operations a user can perform in the folder.

ostate (required) : String

The user's or group's state within the folder (INVITED, MEMBER, FORMER_MEMBER).

odomain (required): String

The Active Directory domain that contains the user's or group's user account.

oaccountType (required): Enum

Identifies whether the returned object is an individual person or a group (USER or GROUP).

ouserInfo (conditional): Object

If a user is returned, this object is also returned. This object lists additional information about the user:

–individual : Boolean

If true, the folder member was invited individually to the folder. If false, the person was invited to the folder as part of a group.

–displayName : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ogroupInfo (required): Object

If a group is returned, this object is also returned. This object lists additional information about the group:

–displayName : String

The group's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.
