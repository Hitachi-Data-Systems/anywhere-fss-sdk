Create team folder

This request converts a private folder into a team folder.

When you share a folder with other people, the folder appears in their HCP Anywhere just as in your own. Changes that occur in the folder occur for all the folder members. For example, if you edit a document in the folder, those edits also occur for all the other members of the folder.

Team folders do not use your personal HCP Anywhere storage. Instead, team folders use space on the HCP Anywhere system set aside by your organization.

You can invite other people to join the team folder using a share/invite request.

Note: Your organization may not have team folders enabled. In this case, a 403 (Forbidden) code is returned in response to this request.

URI

/fss/public/share/team/create

HTTP Method

POST

Parameters

The share/team/create endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the folder to convert into a team folder.

•label (required) : String

The label of the team folder. Once the team folder is created, the label cannot be changed. When a new member is invited to the folder, the name of the folder will be, by default, the folder label.

•contact (required): String

The contact information for the team folder (for example, an email or a phone number). The contact information is available to all the folder members.

•quotaBytes (required): Long

The total storage capacity of the team folder. This needs to be supplied in bytes.

Note: Team folders have a maximum quota capacity, which is set by the HCP Anywhere system administrator.

If the quota of the team folder is set higher than the limit set by the system administrator, then:

•A request is sent to the system administrator for the additional space

•The folder becomes a shared folder.

Example Request

This section contains an example of a share/team/create request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/team/create -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/Group Project", "label": "folder", "contact": "example@example.com", "quotaBytes": 1073741824}'

Example Response

This section contains an example of a response to a successful share/team/create request.

{
    "path": "/Group Project",
    "type": "TEAM",
    "label": "team folder",
    "sync": true,
    "role": "MANAGER",
    "size":13047,
    "teamFolder": {
        "contact":example@example.com",
        "quota":1073741824
    }
}

Description of Response

This section describes the contents of a share/team/create response. You can receive two different responses to this request. The response you receive depends on whether the quota you supplied in the request body was within your organization's team folder quota limits.

If the request is above your organization's team folder quota limit:

• A request for the additional space is sent to your HCP Anywhere system administrator.

•The folder becomes a shared folder.

Response content:

Quota within organization's team folder limits

•path (required) : String

The path to the shared or team folder.

•type (required) : String

The type of shared folder (SHARED or TEAM). For this response, the type is always TEAM.

•label (required) : String

The label of the team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•teamFolder (required): Object

The additional team folder information:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.

Quota above organization's team folder limits

•path (required) : String

The path to the shared or team folder.

•type (required) : Enum

The type of shared folder (SHARED or TEAM). For this response, the type is always SHARED.

•label (required) : String

The label of the shared folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•owner (required): Object

Information about the Owner of the folder. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

ousername: String

The user's username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

odisplayName: String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ouserId: String

The unique identifying string assigned to the user by HCP Anywhere.

• TeamFolderRequest (required): Object

The team folder request information:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
