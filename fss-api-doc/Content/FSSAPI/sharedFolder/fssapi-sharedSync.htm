Enable or disable synchronization

This request enables or disables synchronization of a shared or team folder with a user's desktop application.

You can only enable synchronization of a shared or team folder if you are:

•The Owner of the shared folder

•A Manager of the team folder

•A Collaborator in the shared or team folder

Synchorization for all Mac apps and Windows apps prior to 3.0

When you sync a folder, it appears in your desktop app folder but also uses space on that computer. When you stop syncing a folder, it no longer uses space on your computer and does not appear in the desktop app.

Synchronization for Windows app 3.0 or later

Syncing a folder has no effect. The 3.0 and later Windows desktop app uses the concept of remote and local files for managing space. You can configure whether a file or folder is local (uses space) or remote (does not use space) directly in the desktop app. For more information on doing this, see the HCP Anywhere User Help.

URI

/fss/public/share/updateSettings

HTTP Method

POST

Parameters

The share/updateSettings endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the shared or team folder.

•sync (required) : Boolean

If true, the shared folder and all its contents synchronize with the given user's desktop app folder. If false, the shared folder does not appear in the given user's desktop app folder or use space on the user's computer.

Example Request

This section contains an example of a share/updateSettings request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/updateSettings -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Group Project", "sync": false}'

Example Response

This section contains an example of response to a successful share/updateSettings request.

{
    "path": "/Group Project",
    "type": "SHARED",
    "label": "shared",
    "sync": false,
    "role": "MANAGER",
    "owner" {
        "username": "jsmith",
        "displayName": "John Smith",
        "userId": "8388608"
}

Description of Response

This section describes the contents of a share/updateSettings response.

Response content:

•path (required) : String

The path to the shared or team folder.

•type (required)

The type of shared folder (SHARED or TEAM).

•label (required) : String

The label of the shared or team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•owner (conditional) : String

If the folder is a shared folder, information about the Owner of the folder is returned. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

ousername: String

The user's username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

odisplayName: String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ouserId: String

The unique identifying string assigned to the user by HCP Anywhere.

•teamFolder (conditional): Object

If the folder is a team folder, some additional team folder settings:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
