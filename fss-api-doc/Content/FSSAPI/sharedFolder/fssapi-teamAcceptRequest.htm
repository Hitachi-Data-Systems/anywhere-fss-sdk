Approve team folder request

This request approves a storage request for a team folder.

You must be a Manager of the folder to approve the request. A Manager of a team folder can only approve a storage request for the folder in very specific scenarios. This happens if:

•The system administrator sets the storage capacity of a team folder higher than the storage capacity of a pending request.

•One Manager of a team folder has a lower quota limit than another Manager of the same team folder. For example, one Manager of a team folder can have a quota limit of 10 GB while another Manager has a quota limit of 20 GB. If the Manager with 10 GB requests 15 GB of space for the team folder, the Manager with 20 GB can approve the request.

URI

/fss/public/share/team/approve

HTTP Method

POST

Parameters

The share/team/approve endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the team folder.

Example Request

This section contains an example of a share/team/approve request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/team/approve -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Team Folder"}'

Example Response

This section contains an example of a response to a successful share/team/approve request.

{
    "path": "/Team Folder",
    "type": "TEAM",
    "label": "Team Folder",
    "sync": true,
    "role": "MANAGER",
    "size":13047,
    "teamFolder":
        {

}

Description of Response

This section describes the contents of a share/team/approve response.

Response content:

•path (required) : String

The path to the shared or team folder.

•type (required) : String

The type of shared folder (SHARED or TEAM). For this response, the type is always TEAM.

•label (required) : String

The label of the team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•teamFolder (required): Object

The additional team folder information:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
