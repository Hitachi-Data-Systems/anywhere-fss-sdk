Update team folder settings

This request updates the settings of a team folder.

With this endpoint, you can change the:

•Contact information for the team folder

•Total storage capacity of the team folder

To change the folder settings, you need to be a Manager in the folder.

URI

/fss/public/share/team/settings/update

HTTP Method

POST

Parameters

The share/team/settings/update endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•path (required) : String

The path to the team folder.

•contact (optional): String

The new contact information for the team folder (for example, an email or a name). The contact information is available to all the folder members.

•quotaBytes (optional): Long

The total storage capacity of the team folder. This needs to be supplied in bytes.

Note: Team folders have a maximum quota capacity, which is set by the system administrator.

If the quota of the team folder is set higher than the limit set by the system administrator, a request is sent to the system administrator for the additional space

Example Request

This section contains an example of a share/team/settings/update request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/team/settings/update -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Group Project","contact": "NewEmail@example.com", "quotaBytes": "1073741830"}'

Example Response

This section contains an example of a response to a successful share/team/settings/update request.

{
    "path": "/Group Project",
    "type": "TEAM",
    "label": "team folder",
    "sync": true,
    "role": "MANAGER",
    "teamFolder": {
        "contact":"NewEmail@example.com",
        "quota":1073741830
    }
}

Description of Response

This section describes the contents of a share/team/settings/update response.

If you adjusted the quota as part of this request and the quota was above your organization's team folder quota limits, a request for the additional space is sent to your HCP Anywhere system administrator. In this case, information about the request is returned in the response body along with information about the team folder.

Response content:

•path (required) : String

Path to the team folder.

•type (required) : String

The type of shared folder (SHARED or TEAM). For this response, the type is always TEAM.

•label (required) : String

The label of the team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•teamFolder (conditional) or TeamFolderRequest: Object

If the quota is within your organization's team folder quota limits, the response is "teamFolder". If the quota is above your organization's team folder quota limits, the response is "TeamFolderRequest". Both "teamFolder" and "TeamFolderRequest" return this information:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
