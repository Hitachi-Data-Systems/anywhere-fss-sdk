List collaborative folders

This request lists a user's shared and team folders.

URI

/fss/public/share/list

HTTP Method

POST

Parameters

There are no parameters for a share/list request.

Example Request

This section contains an example of a share/list request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/list -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0"

Description of Response

This section describes the contents of a share/list response.

Response content:

•sharedFolders (required): String

A list of the user's shared and team folders:

opath (required) : String

The path to the shared or team folder.

otype (required):Enum

The type of collaborative folder (SHARED or TEAM).

olabel (required) : String

The label of the shared or team folder.

osync (required): Boolean

If true, the shared folder and all its contents are being synchronization with your HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in your desktop application folder and is downloaded to your computer. If false, the shared folder does not appear in your desktop application folder or use space on your local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

orole (required) : String

The role of the folder. The folder role is always "MANAGER".

oowner (conditional) : String

If the folder is a shared folder, information about the Owner of the folder. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

–username (required) : String

The user's username. The username is acquired from the authentication provider (SAML or AD) of that user.

–displayName (required) : String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of that user.

–userId: String

A unique identifying string assigned to the user by HCP Anywhere.

osize (required): String

The size of the folder in bytes.

oteamFolder (conditional): Object

If the item is a team folder or within a team folder, some additional team folder information:

–contact: String

The contact of the team folder.

–quota: Long

The total storage capacity of the team folder. This is returned in bytes.
