List folder alerts

This request lists the collaborative folder alerts for a user. Collaborative folder alerts include:

•Changes of member role in a folder

•Changes of quota in a team folder

•The administrator adding or deleting mobilized data shares from the user's HCP Anywhere account

URI

/fss/public/share/alerts

HTTP Method

POST

Parameters

The share/alerts endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•dismiss (required): Boolean

If true, the alerts are returned and then cleared. If false, the alerts are returned but are not cleared.

Example Request

This section contains an example of a share/alerts request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anyhwere.example.com/fss/public/share/alerts -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"dismiss": false}'

Example Response

This section contains an example of response to a successful share/alerts request.

{
    "alerts":[
       {
          "fsId":1729382256910270467,
          "label": "Sales Team",
          "alert": "User jsmith had their role for shared folder with label Sales_Reports updated by user tlomas"
      }
   ]
}

Description of Response

This section describes the contents of a share/alerts response.

Response content:

•alerts (required) : Object

The collaborative folder alerts:

ofsId (required) : Long

A unique number assigned to the shared or team folder by HCP Anywhere.

olabel (required) : String

The label of the folder.

oalert (required): String

The alert message.

oteamFolderInfo (conditional): Object

If the item is a team folder, some additional team folder information:

–contact: String

The contact of the team folder.

–quota: Long

The total storage capacity of the team folder. This is returned in bytes.
