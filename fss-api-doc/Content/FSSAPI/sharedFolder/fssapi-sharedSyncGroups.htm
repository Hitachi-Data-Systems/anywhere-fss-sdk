Update group membership

This request updates the groups in a shared or team folder.

With this request, all the groups in the shared or team folder are updated to reflect their current membership in Active Directory. By default, groups are only updated at the end of each day as opposed to every time a person is added or deleted from one of the groups in Active Directory.

URI

/fss/public/share/syncMembership

HTTP Method

POST

Parameters

The share/syncMembership endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•path (required): String

The path to the shared or team folder.

•increaseRole (required): Boolean

If true, the role of each group member is changed to reflect the role assigned to their group, except when the group member has a role that is more permissive than the group. Permissiveness goes in this order, from lowest to highest: Viewer, Collaborator, Manager.

If false, the role of all group members stays the same.

•inviteDeclined (required): Boolean

If true, any member of the group that has declined to join, left, or been deleted from the shared folder is reinvited to the folder.

Example Request

This section contains an example of a share/syncMembership request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/share/syncMembership -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"path": "/Sales Team", "increaseRole": false, "inviteDeclined": false}'

Description of Response

This section describes the contents of a share/syncMembership response.

Response content:

•path (required) : String

The path to the shared or team folder.

•type (required)

The type of shared folder (SHARED or TEAM).

•label (required) : String

The label of the shared or team folder.

•sync (required): Boolean

If true, the collaborative folder and all its contents are being synchronization with the user's HCP Anywhere desktop application. If a folder is synchronized, the shared folder appears in their desktop application folder and is downloaded to your computer. If false, the collaborative folder does not appear in their desktop application folder or use space on their local hard drive.

For more information on synchronization, see the share/updateSettings endpoint.

•role (required) : String

The role of the folder. The folder role is always "MANAGER".

•size (required): String

The size of the folder in bytes.

•owner (conditional) : String

If the folder is a shared folder, information about the Owner of the folder is returned. The person who creates the shared folder is the Owner of the folder. The Owner of the folder manages folder membership.

ousername: String

The user's username. The username is acquired from the identity provider (SAML or AD) of the HCP Anywhere system.

odisplayName: String

The user's display name. The display name is acquired from the authentication provider (SAML or AD) of the HCP Anywhere system.

ouserId: String

The unique identifying string assigned to the user by HCP Anywhere.

•teamFolder (conditional): Object

If the folder is a team folder, some additional team folder settings:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.
