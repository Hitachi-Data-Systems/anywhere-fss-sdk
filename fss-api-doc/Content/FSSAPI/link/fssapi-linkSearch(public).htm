Search link contents (public)

This request searches the contents of folder that was accessed through a link.

Notes:  

•You can use this endpoint to access a public link. Public links are accessible without login credentials. Because of this, you do not have to include an authentication token in this request.

•You cannot use this endpoint to access an internal link.  Internal links are only accessible with valid HCP Anywhere login credentials. To search through the contents of a file or folder that was accessed through a public link, you need to use a link/private/browse/search request.

•The person who creates the link sets whether the link is internal or public. These settings are part of the link metadata. The link metadata is returned in response to a number of the link requests.

URI

/fss/public/link/private/browse/search

HTTP Method

POST

Parameters

The link/public/browse/search endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•path (required) : String

If the link points to a folder, enter "/" to search through the contents of the entire folder. To search through a particular subdirectory within that folder, enter the path to that subdirectory.

•itemName (required) : String

The name of the file or folder where the link is pointing.

•linkToken (required) : String

A unique string assigned to the link by HCP Anywhere. The link token is part of the link metadata and is returned in response to a number of link requests.

•accessCode (conditional): String

An additional code assigned to the link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

•filter (required): String

The string which is searched for in the folder. Files and folders that match or partially match the string are returned. The string needs be at least 3 characters long.

Example Request

This section contains an example of a link/public/browse/search request in cURL. Since this request accesses a public link, an authentication token is not required in this request.

curl -kiX POST https://anywhere.example.com/fss/public/link/public/browse/search -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"linkToken": "hh3eGZSd3hyUHd4Z", "itemName": "Sales Reports", "path": "/", "filter": "Quarter1"}'

Description of Response

This section describes the contents of a link/public/browse/search response.

Response content:

•entries (required) : Object

A list of items that match the search criteria.

otype (required) : Enum

The type of item (FILE or FOLDER).

oname (required) : String

The name of the file or folder.

ochangeTime (required) : Long

The time at which the item was last accessed or modified. This time is returned in Unix format.

osize (conditional) : Long

If the item is a file, the size of the file in bytes.

ohash (conditional) : String

If the item is a file, the hex encoded sha-384 hash of the file.

oaccess (required) : String

The type of operations the given user can perform on the item:

–NO ACCESS — The person cannot access the file or folder.

–VIEWER — The person can view and download the file or folder. However, if the item is a folder, the person cannot upload files to the folder.

–COLLABORATOR — The person can download, upload, move, and delete files and folders.

oetag (required) : String

The etag of the item. The etag identifies the version of the file or folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

ostate (required) : String

The state of the item (CREATE or DELETE).

osharing (conditional): Object

If the item is a shared folder, team folder, or mobilized share or within one of these, some additional folder information.

–canUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

–isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized shares are mount points, although the items within them are not.

–label (conditional) : String

If the item is a shared or team folder, the label of the folder.

–type:Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

–teamFolder (conditional): Object

If the item is a team folder or within a team folder, some additional team folder information:

•contact: String

The contact of the team folder.

•quota: Long

The total storage capacity of the team folder. This is returned in bytes.

obackup (conditional): Object

If the item is a backup file or folder, some additional backup information:

–isActive: Boolean

If true, the backup folder is connected to the HCP Anywhere desktop application. 

The backup folder can be disconnected from the desktop application if the desktop application is deregistered or if the backup feature is disabled by the system administrator.

–isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. The backup parent folder, which is automatically created when the desktop application is installed with backup enabled, is always a mount point.

–operatingSystem: Enum

The type of device on which the desktop application is installed (WINDOWS, WINDOWS NSC, or MAC). For version 3.0 or later of the Windows desktop application, the operatingSystem is "WINDOWS NSC".

–hasAssociatedClient: Boolean

Each backup file or folder is associated with the desktop application that performed its backup. If true, the desktop application associated with the backup folder is currently registered. If false, the desktop application associated with the backup folder is currently deregistered.

Note: An application can be deregistered if:

•You deregister it in the User Portal in the My Devices window

• Your administrator deregisters it in the HCP Anywhere Management Console

Additional Notes

If an access code was used in the request, a cookie is returned in response header. That header looks like this:

Set-Cookie: AW-Access-Code=1f04e79f3c8;Version=1;Path=/;Max-Age=600


The cookie can be attached to subsequent link requests where otherwise the access code would be required.
