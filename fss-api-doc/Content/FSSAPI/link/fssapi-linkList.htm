List links

This request lists all the existing links for a user.

URI

/fss/public/link/list

HTTP Method

POST

Parameters

The link/list endpoint can contain request body parameters. However, you do not need to provide any parameters in the request body. If you are not passing in any parameters, you need to include an empty JSON body in the request.

Request body parameters:

•pageSize (optional) : Integer

The maximum number of items to return in the request. The default is 100.

•filter (optional): String

The search criteria. Files and folders that match or partially match the string are returned. The string needs be at least 3 characters long. If this parameter is not supplied, all the files are folders are returned.

•sortingCriterion (optional): Enum

The way in which the links are sorted in the response body (NAME, EXPIRATION, PERMISSIONS, PUBLIC, HAS_ACCESS_CODE). If you enter "NAME", the links are sorted alphabetically by name from A-Z. If you enter "PERMISSIONS", the links are also sorted by name but for each file the public links to the file are listed first, followed by the upload links, followed by the links with both permissions. If you enter "PUBLIC", the public links are listed first, followed by the files with internal links. If you enter "HAS_ACCESS_CODE", the links with an access code are listed first, followed by the links without an access code.

The default is "NAME". The way in which the links are sorted also depends on the "sortingDirection" parameter.

•sortingDirection (optional): Enum

The direction in which the links are sorted. To ascend through the list of links, enter "ASC". To descend through the list of links, enter "DESC".

For example, if "NAME" is supplied for the "sortingCriterion" parameter and "ASC" is supplied for this parameter, the links are returned in alphabetical order from A-Z. If "NAME" is supplied for the "ordering" parameter and "DESC" is supplied for this parameter, the items are sorted alphabetically from Z-A.

The default is "ASC".

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of you are.

Example Request

This section contains an example of a link/list request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/list -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" - d '{"pageSize": 20}'

Description of Response

This section describes the contents of a link/list response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of links you are. You can supply this page token in the body of a request to return the next set of links.

•links (required) : Object

A list of the user's active links:

ourl (required) : String

The actual link.

opath (required) : String

The path to the file or folder where the link is pointing.

oexpirationDate (required) : Long

The time at which the link expires. This time is returned in Unix format.

opublic (required) : Boolean

If true, the link is publicly accessible (accessible without first entering login credentials). If false, the link is internally accessible (only accessible by other HCP Anywhere users in the network).

oaccessible (required) : Boolean

If true, the link is active. If false, the link is inactive. A link becomes inactive if the file or folder that the link is points to has been moved, renamed, or deleted or link sharing has been disabled by the HCP Anywhere administrator.

oaccessCode (conditional) : String

An additional code assigned to the link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

opermissions (required) : Enum

The type of operations a person can perform on the file or folder when accessed through the link:

–READ — The user can view and download files.

–UPLOAD — If the link is to a folder, the user can upload files to the folder.

otoken (required) : String

A unique string assigned to the link by HCP Anywhere. The link token is supplied as a parameter in some of the link requests.

otype (required) : Enum

The type of item (FILE or FOLDER).

oitemName (required) : String

The name of the file or folder that the links points to.

otype (required) : Enum

The type of item (FILE, FOLDER, or BROKEN). If the item that the link points to has been moved, renamed, or deleted, the type is BROKEN.
