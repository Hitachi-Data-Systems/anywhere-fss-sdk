Save files to HCP Anywhere (internal)

This request saves files and folders accessed through a link to HCP Anywhere.

You can save:

•A file that a link points directly to.

•Files and folders within a folder that a link points to.

Notes:  

•You can use this endpoint to access an internal link. Internal links are only accessible with valid HCP Anywhere login credentials.

•You cannot use this endpoint to access a public link. Public links are accessible without login credentials. To list the link settings of a public link, you need to use a link/public/copyToLocal request.

•The person who creates the link sets whether the link is internal or public. These settings are part of the link metadata. The link metadata is returned in response to a number of the link requests.

URI

/fss/public/link/private/copyToLocal

HTTP Method

POST

Parameters

The link/private/copyToLocal endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•itemName (required): String

The name of the file or folder where the link is pointing.

•linkToken (required) : String

A unique string assigned to the link by HCP Anywhere. The link token is part of the link metadata and is returned in response to a number of link requests.

•destinationPath (required): String

The path in HCP Anywhere where the files are copied to.

•accessCode (conditional): String

An additional code assigned to the link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

•pathList (optional): Array

The path to each item that is being copied to HCP Anywhere.

Example Request

This section contains an example of a link/private/copyToLocal request in cURL. In this example, the link points to a folder called "Sales Team" and the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/private/copyToLocal -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"itemName": "Sales Team", "linkToken": "MF77byLe-2giXvtv", "pathList": ["Q1-Reports.docx","Q2-Reports.docx"], "destinationPath": "/Sales Reports"}'

Example Response

This section contains an example of a response to a successful link/private/copyToLocal request.

{
    "successfulCopies": [
      {
          "path": "/Q1-Report.docx"
      }
   ],
    "failedCopies": [
      {
          "path": "/Q2-Report.docx",
          "errorMsg": "The destination folder already contains a file or folder with the same name."
      }
   ]
}

Description of Response

This section describes the contents of a link/private/copyToLocal response.

Response content:

•SuccessfulCopies (conditional): Object

A list of the items that were successfully copied to HCP Anywhere:

opath (required): String

The path where the file or folder was copied from.

•failedCopies (conditional): Object

A list of items that failed to be copied to HCP Anywhere:

opath (required): String

The path where the file or folder failed to be copied from.

oerrorMsg (required): String

A description of why the file or folder could not be copied to HCP Anywhere.

Additional Notes

If an access code was used in the request, a cookie is returned in response header. That header looks like this:

Set-Cookie: AW-Access-Code=1f04e79f3c8;Version=1;Path=/;Max-Age=600


The cookie can be attached to subsequent link requests where otherwise the access code would be required.
