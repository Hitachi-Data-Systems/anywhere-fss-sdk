Create link

This request creates a link to a file or folder.

URI

/fss/public/link/create

HTTP Method

POST

Parameters

The link/create endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the file or folder where the link will point.

•accessCode (required) : Boolean

If true, an additional access code is created for the link. In this case, to access the link, a person must have both the link and the access code.

•permissions (required) : String

The type of operations a person can perform on the file or folder when accessed through the link:

oREAD — The user can view and download files.

oUPLOAD — If the link points to a folder, the user can upload files to the folder.

•expirationDays (optional) : Integer

The number of days until the link expires. The settings on the Management Console determine the default number of days until the link expires.

•public (optional) : Boolean

If true, the link is publicly accessible (accessible without first providing login credentials). If false, the link is internally accessible (accessible only by other HCP Anywhere users in the network). The settings on the Management Console determine the default accessibility of a link (public or internal).

Example Request

This section contains an example of a link/create request in cURL. In this example, the link points to a folder called "Team-Projects" and the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/create -H "Accept: application/json" -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"path": "/Team-Projects", "expirationDays": 30, "public": true, "accessCode": true, "permissions": ['READ']}'

Example Response

This section contains an example of a response to a successful link/create request. In an actual response, the etags are much longer.

{
    "url": "https://anywhere.example.com/u/79KZnP1SmZT90pmc/Work-Projects?l",
    "path": "/Work Projects",
    "expirationDate": 1459440864685,
    "public": true,
    "accessible":true,
    "accessCode": "8h;nDy0z",
    "permission": ["READ"],
    "token": "79KZnP1SmZT90pmc",
    "type":"FOLDER",
    "itemName": "Work Projects"
}

Description of Response

This section describes the contents of a link/create response.

Response content:

•url (required) : String

The actual link.

•path (required) : String

The path to the file or folder that the link points to.

•expirationDate (required) : Long

The time at which the link expires. This time is returned in Unix format.

•public (required) : Boolean

If true, the link is publicly accessible (accessible without first entering login credentials). If false, the link is internally accessible (accessible only by other HCP Anywhere users in the network).

•accessible (required) : Boolean

If true, the link is active. If false, the link is inactive. A link becomes inactive if the file or folder that the link is points to has been moved, renamed, or deleted or link sharing has been disabled by the HCP Anywhere administrator.

•accessCode (conditional) : String

An additional code assigned to the given link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

•permissions (required) : String

The type of operations that a person can perform on the file or folder when accessed through the link:

oREAD — The user can view and download files.

oUPLOAD — If the link is to a folder, the user can upload files to the folder.

•token (required) : String

The unique string assigned to the link by HCP Anywhere. The link token is supplied as a parameter in some of the other link endpoints.

•type (required) : Enum

The type of item (FILE or FOLDER).

•itemName (required) : String

The name of the file or folder that the links points to.

Additional Notes

If an access code was used in the request, a cookie is returned in response header. That header looks like this:

Set-Cookie: AW-Access-Code=1f04e79f3c8;Version=1;Path=/;Max-Age=600


The cookie can be attached to subsequent link requests where otherwise the access code would be required.
