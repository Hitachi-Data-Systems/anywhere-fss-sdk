Check link access code (internal)

This request checks whether an access code is valid for a link.

People can generate a new access code for one of their existing links. When a person generates a new access code for the link, the link remains the same. However, the new access code is required to access the link while the old access code becomes invalid.

Notes:  

•You can use this endpoint to access an internal link. Internal links are only accessible with valid HCP Anywhere login credentials.

•You cannot use this endpoint to access a public link. Public links are accessible without login credentials. To check if an access code is valid for a public link, you need to use a link/public/checkAccess request.

•The person who creates the link sets whether the link is internal or public. These settings are part of the link metadata. The link metadata is returned in response to a number of the link requests.

URI

/fss/public/link/private/checkAccess

HTTP Method

POST

Parameters

The link/private/checkAccess endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•linkToken (required) : String

A unique string assigned to the link by HCP Anywhere. The link token is part of the link metadata and is returned in response to a number of link requests.

•itemName (required): String

The name of the file or folder where the link is pointing.

•accessCode (required): String

An additional code assigned to the given link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

Example Request

This section contains an example of a link/private/checkAccess request in cURL. In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/private/checkAccess -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"linkToken": "XFwAIE7cACdOXAAg", "itemName": "Marketing Agenda.docx", "accessCode": ")X8Q3hNb"}'

Example Response

If the access code is valid, no information is returned in the response body.

If the access code is invalid, an 403 (Invalid_Access_Code) is returned in response to the request.

Additional Notes

If the access code is valid, a cookie is returned in response header. That header looks like this:

Set-Cookie: AW-Access-Code=1f04e79f3c8;Version=1;Path=/;Max-Age=600


The cookie can be attached to subsequent link requests where otherwise the access code would be required.
