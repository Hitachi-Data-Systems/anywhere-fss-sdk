List link settings (internal)

This request lists the settings of a link. These settings include:

•The access permissions for the link (READ or UPLOAD).

•The type of item that the link points to (FILE or FOLDER).

•Whether or not the link requires an additional code to access.

Notes:  

•You can use this endpoint to access an internal link. Internal links are only accessible with valid HCP Anywhere login credentials.

•You cannot use this endpoint to access a public link. Public links are accessible without login credentials. To list the link settings of a public link, you need to use a link/public/info request.

•The person who creates the link sets whether the link is internal or public. These settings are part of the link metadata. The link metadata is returned in response to a number of the link requests.

URI

/fss/public/link/private/info

HTTP Method

POST

Parameters

The link/private/info endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•itemName (required) : String

The name of the file or folder where the link is pointing.

•linkToken (required) : String

A unique string assigned to the link by HCP Anywhere. The link token is part of the link metadata and is returned in response to a number of link requests.

Example Request

This section contains an example of a link/private/info request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/private/info -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"itemName": "Sales Overview.pdf", "linkToken": "YK9JjXIvSYlyr0mN" }'

Example Response

This section contains an example of a response to a successful link/private/info request.

{
    "requiresAccessCode":false,
    "permissions":["READ"],
    "type":"FILE",
    "contentType": "PDF",
    "size":32288
}

Description of Response

This section describes the contents of a link/private/info response.

Response content:

•requiresAccessCode (required): Boolean

If true, the file or folder that the link points to cannot be accessed without the link's access code.

•permissions (required) : Enum

The type of operations a person can perform on the file or folder when accessed through the link:

oREAD — The user can view and download files.

oUPLOAD — If the link is to a folder, the user can upload files to the folder.

•type (required) : Enum

The type of item (FILE, FOLDER, or BROKEN). If the item that the link points to has been moved, renamed, or deleted, the type is BROKEN.

•contentType (conditional) : String

If the item is a file, the format of the file is returned (for example, "audio/mp3", or "image/jpeg"). If the file is an audio or video file and over 5 MB, not a recognized file format, or link preview is disabled by the HCP Anywhere administrator, the contentType is always "application/octet-stream". If the item is a folder, the contentType field is not returned in the response body.

•size (conditional) : Long

If the item is a file, the size of the file in bytes.
