Update link

This request changes the access code, expiration date, or both the access code and expiration date of an existing link.

URI

/fss/public/link/update

HTTP Method

POST

Parameters

The link/update endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•path (required) : String

The path to the file or folder where the link is pointing.

•url (required) : String

The actual link.

•updateAccessCode (required) : Boolean

If true, a new access code is created for the link. In this case, the old access code becomes invalid.

•expirationDate (optional) : Long

The new time at which the link expires. This time needs to be supplied in Unix millisecond format.

Example Request

This section contains an example of a link/update request in cURL. In this request, the authentication token is "abc=". In an actual request, the authentication token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/link/update -H "Content-Type: application/json" -H "Authorization: Bearer abc=" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d '{"path": "/Sales_Overview.pptx", "url": "https://anywhere.example.com/u/jFv48p7b-PeeW_jy/Sales_Overview.pptx?l", "updateAccessCode": false, "expirationDate": 1467831494000}'

Example Response

This section contains an example of a response to a successful link/update request.

{
    "url":"https://anywhere.example.com/u/jFv48p7b-PeeW_jy/Sales_Overview.pptx?l",
    "path": "/Sales_Overview.pptx",
    "expirationDate": 1467831494000,
    "public": true,
    "accessible":true,
    "permissions": ["READ"],
    "token": "jFv48p7b-PeeW_jy",
    "type": "FILE",
    "itemName": "Sales_Overview.pptx"
}

Description of Response

This section describes the contents of a link/update response.

Response content:

•url (required) : String

The actual link.

•path (required) : String

The path to the file or folder that the link points to.

•expirationDate (required) : Long

The time at which the link expires. This time is returned in Unix format.

•public (required) : Boolean

If true, the link is publicly accessible (accessible without first entering login credentials). If false, the link is internally accessible (accessible only by other HCP Anywhere users in the network).

•accessible (required) : Boolean

If true, the link is active. If false, the link is inactive. A link becomes inactive if the file or folder that the link is points to has been moved, renamed, or deleted or link sharing has been disabled by the HCP Anywhere administrator.

•accessCode (conditional) : String

An additional code assigned to the given link by HCP Anywhere. The person who creates the link decides if the link requires this additional code.

•permissions (required) : String

The type of operations that a person can perform on the file or folder when accessed through the link:

oREAD — The user can view and download files.

oUPLOAD — If the link is to a folder, the user can upload files to the folder.

•token (required) : String

The unique string assigned to the link by HCP Anywhere. The link token is supplied as a parameter in some of the other link endpoints.

•type (required) : Enum

The type of item (FILE or FOLDER).

•itemName (required) : String

The name of the file or folder that the links points to.
