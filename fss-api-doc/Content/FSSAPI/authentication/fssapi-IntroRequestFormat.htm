Request format

In general, the File Sync and Share API uses HTTP POST requests with JSON arguments and JSON responses.

In each request, you need to include four headers:

•Authorization

•Content-Type

•Accept-Type

•X-HCPAW-FSS-API-VERSION

Authorization header

The Authorization header contains an authentication token. The authentication token specifies the user making the request.

You can acquire an authentication token using a login/oauth request. This request is for Active Directory authentication.

Note: The File Sync and Share API does not support SAML authentication.

Content-Type header

The Content-Type header specifies the format of the data supplied in the body of the request you're making. For most requests, the Content-Type is application/json. However, there are some exceptions to this. For example, basic authentication uses an application/x-www-form-urlencoded Content-Type header while both upload and update use an appliction/octet-stream Content-Type header.

For the endpoints that use a JSON request body, the Content-Type header looks like this in cURL:

-H "Content-Type: application/json"

Accept-Type header

The Accept-Type header specifies the format of the data that is accepted in response to the request you're making. For most requests, the Accept-Type header is application/json. However, there are some exceptions to this. For example, the download file endpoints accept application/octet-stream in response to a request.

For the endpoints that accept JSON, the Accept-Type header looks like this in cURL:

-H "Accept: application/json"

X-HCPAW-FSS-API-VERSION header

The X-HCPAW-FSS-API-VERSION header specifies the File Sync and Share API version that you are using. The File Sync and Share API version does not have to be the same as the HCP Anywhere server. The HCP Anywhere server supports older versions of the File Sync and Share API. Version 2.1.1 is the oldest supported File Sync and Share API version .

For an HCP Anywhere server at version 3.0.3, the X-HCPAW-FSS-API-VERSION header looks like this in cURL:

-H "X-HCPAW-FSS-API-VERSION: 3.0.3"

URL Request Structure

For each File Sync and Share API endpoint, the URL has the same structure. The URL begins like this:

https://anywhere.example.com/fss/public

This is followed by the URI of the endpoint you are using. For example, the URL for an upload new file request looks like this in cURL:

https://anywhere.example.com/fss/public/file/stream/create

In an actual request, “anywhere.example.com” should be replaced with the server that the request is being made to.
