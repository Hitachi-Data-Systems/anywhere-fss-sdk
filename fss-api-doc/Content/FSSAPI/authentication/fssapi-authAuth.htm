Active Directory authentication

This request authenticates a user and returns an authentication token.

Once you have an authentication token, you can use the other File Sync and Share API endpoints by including the authentication token and any other required parameters of that particular endpoint in a request. The authentication token is supplied as a header in each request. For more information, see Authentication overview.

URI

/fss/public/login/oauth

HTTP Method

POST

Parameters

The login/oauth endpoint requires only request body parameters for both form and JSON authentication. The request body needs to be in JSON format.

Request body parameters for basic authentication:

•grant_type (required) : String

The grant type needs to be this string: "urn:hds:oauth:negotiate-client".

Request body parameters for form authentication:

•username (required) : String

The username of the user.

•password (required) : String

The password of the given user.

•grant_type (required) : String

The grant type needs to be this string: "urn:hds:oauth:negotiate-client".

Example Request

This section contains an example of a basic and form login/oauth request in cURL. In these two examples, the username is "jsmith" and the password is "mypassword".

Form authentication:

curl https://anywhere.example.com/fss/public/login/oauth -iku "jsmith:mypassword" -H "Content-Type: application/x-www-form-urlencoded" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -d 'grant_type=urn:hds:oauth:negotiate-client'


JSON authentication:

curl -kiX POST https://anywhere.example.com/fss/public/login/oauth -H "X-HCPAW-FSS-API-VERSION: 3.0.3" -H "Accept: application/json" -H "Content-Type: application/json" -d '{"username": "jsmith", "password": "mypassword", "grant_type": "urn:hds:oauth:negotiate-client"}'

Example Response

This section contains an example of a response to a successful login/oauth request. In an actual response, the access token and hash are much longer.

{
    "access_token": "abc=",
    "token_type": "Bearer",
    "expires_in": 2592000
}

Description of Response

This section describes the contents of a login/oauth response.

Response content:

•access_token (required) : String

An authentication token, which holds the login credentials of the user who made the request.

•token_type (required) : String

The type of token. The type of token is always Bearer as that is the only type the File Sync and Share API supports.

•expires_in (required) : Integer

The amount of time (in seconds) before the access token expires.
