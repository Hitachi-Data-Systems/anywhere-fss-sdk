SAML authentication

This request redirects to the URL of a SAML identity provider. Once the request redirects to a SAML identity provider, that SAML identity provider is responsible for authenticating the user.

URI

/fss/public/saml/redirect

HTTP Method

POST

Parameters

The saml/redirect endpoint requires only request body parameters. The request body needs to be in JSON format.

Request body parameters:

•id (required): String

The unique number assigned to the SAML server by HCP Anywhere. You can acquire the id of a SAML server for an HCP Anywhere system using a provider/list request.

•location (optional): String

The URL of the SAML server. If this parameter is not included in the request, the URL specified on the SAML server is used.

•clientType (optional): Enum

The type of client making the saml/redirect request (PORTAL or CLIENT). If this request is being made through a web browser, the clientType is PORTAL.

Example Request

This section contains an example of a saml/redirect request in cURL.

curl -kiX POST https://anywhere.example.com/fss/public/saml/redirect -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 3.0.0" -d '{"id": "2305843009213693954", "clientType": "PORTAL", "location": "https://anywhere.com/mobile/saml/redirect"}'

Example Response

For a successful saml/redirect request, no information is returned in the response body.

Additional Notes

For a successful saml/redirect request, a 302 (Found) response code is returned.
