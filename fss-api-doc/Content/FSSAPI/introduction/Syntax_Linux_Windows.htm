Syntax for Linux and Windows

For any File Sync and Share API request that requires a JSON request body, the request body syntax in cURL is slightly different on Linux and Windows.

On a Linux-based terminal (e.g. Linux terminal or Cygwin), the request body needs to be surround by single quotes ('). A request body should look something like this on Linux:

-d '{"path": "/Work-Projects/Sales-Reports", "createParents": true}'

On Windows-based terminal (e.g. Windows command prompt), the request body needs to be surrounded by double quotes (") and any parameters in the request body need to be surrounded by double quotes (") that have been escaped with a forward slash. A request body should look something like this on Windows:

-d "{/"path/": /"/Work-Projects/Sales-Reports/", /"createParents/": true}"
