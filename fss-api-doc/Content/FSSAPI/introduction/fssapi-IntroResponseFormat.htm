Response format

The table below describes the headers returned in response to a File Sync and Share API request.

Response header Description

X-HCPAW-SUPPORTED-FSS-API-VERSIONS

 The File Sync and Share API versions supported by the HCP Anywhere server that the request was made to.

X-HCPAW-FSS-API-VERSION

 The version of the HCP Anywhere server that the request was made to.
X-HCPAW-System-Id The HCP Anywhere system id, which is always the global unique identifier of the system.
Content-Type The type of content returned in the response. Typically, the Content-Type is "application/json".
Set-Cookie

The Set-Cookie header is returned only if:

•You are making a request using one of the link endpoints

•The request body contains an access code

When returned, the Set-Cookie response header contains a cookie. The cookie can be attached to subsequent link requests where otherwise the access code would be required.

The response body content that each Data Management API request returns is described in the topic for each endpoint.
