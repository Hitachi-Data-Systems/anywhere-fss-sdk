Response codes

The table below describes the most common codes returned in response to aFile Sync and Share API request.

HTTP Response Code Error string Meaning
200 OK

The operation was successful.

 
201 Created The create was successful. This response code is returned for operations such as a file or folder create.
204 OK The delete was successful.
400 Bad Request The request contains an invalid parameter. See the error message in the response body for more information.
401 Unauthorized The request contains an invalid or expired access token.
403 Client Not Allowed

The File Sync and Share API is not enabled for the user making the request. The File Sync and Share API can be enabled in the HCP Anywhere Management Console by an HCP Anywhere administrator.

For more information, see Enabling the File Sync and Share API.
404 Not Found The item specified in the request was not found in HCP Anywhere. Typically, this means that the URI supplied in the request does not exist.
409 Conflict A conflict was encountered while trying to perform the request. Conflicts occur when a single file is modified from multiple places at once. In this case, HCP Anywhere accepts the changes that it receives first and creates a conflict file for the changes it receives second. For more information on conflicts, see the HCP Anywhere User Help.
413 Request Entity Too Large An HCP Anywhere limit was encountered while trying to perform the request. See the error message in the response body for more information.
500 Internal Server Error An internal server error occurred.
503 Service Unavailable The HCP Anywhere server is temporarily unable to perform the request. Try the request again, gradually increasing the delay between each successive attempt. If this error persists, contact your HCP Anywhere administrator.
