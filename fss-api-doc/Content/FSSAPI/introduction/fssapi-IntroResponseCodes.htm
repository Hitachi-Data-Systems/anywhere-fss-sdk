Response codes

The table below describes the most common codes returned in response to a File Sync and Share API request.

HTTP Response Code Error string Meaning
200 OK

The operation was successful.

 
204 OK The delete was successful.
400 Bad Request The request contains an invalid input parameter. See the error message in the response body for more information.
401 Unauthorized The request contains an invalid or expired authentication token.
403 Client Not Allowed

The File Sync and Share API is not enabled for the user making the request. The File Sync and Share API can be enabled in the HCP Anywhere Management Console by an HCP Anywhere administrator.

For more information, see Enabling the File Sync and Share API.
404 Not Found The item specified in the request was not found in HCP Anywhere. Typically, this means that the path supplied in the request does not exist.
409 Conflict A conflict was encountered while trying to perform the request.
413 Request Entity Too Large An HCP Anywhere limit was encountered while trying to perform the request. See the error message in the response body for more information.
500 Internal Server Error An internal server error occurred.
503 Service Unavailable The server is temporarily unable to perform the request. Try the request again, gradually increasing the delay between each successive attempt. If this error persists, contact your HCP Anywhere administrator.
