Authentication overview

To make a successful request with most File Sync and Share API endpoints, you need to include an authentication token in the request. The authentication token identifies the user making the request and holds their login credentials.

You can acquire an authentication token using one of these endpoints:

•The login/oauth endpoint for Active Directory authentication

•The saml/redirect endpoint for SAML authentication

If the request is successful, an authentication token is returned in the response body.

Once you have an authentication token, you can use the other File Sync and Share API endpoints by supplying the authentication token and any other required parameters of that particular endpoint. The authentication token is supplied as a header in each request. In cURL, the authorization header looks like this:

-H "Authorization: Bearer abc="

In an actual request, the authentication token is much longer than the example “abc=” authentication token used above.
