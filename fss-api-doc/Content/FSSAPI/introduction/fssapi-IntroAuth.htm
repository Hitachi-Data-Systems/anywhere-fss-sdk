Authentication overview

To make a successful request with most File Sync and Share API endpoints, you need to include an access token in the request. The access token identifies the user making the request and holds their login credentials.

You can acquire an access token using a login/oauth request. The File Sync and Share API only supports authentication using an Active Directory (AD) user. The File Sync and Share API does not support SAML authentication.

If the login/oauth request is successful, an access token is returned in the response body.

Once you have an access token, you can use the other File Sync and Share API endpoints by supplying the access token and any other required parameters of that particular endpoint. The access token is supplied as a header in each request. In cURL, the authorization header looks like this:

-H "Authorization: Bearer abc="

In an actual request, the access token is much longer than the example "abc=" access token used above.
