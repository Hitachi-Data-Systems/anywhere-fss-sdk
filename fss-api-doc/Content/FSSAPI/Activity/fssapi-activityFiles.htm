List file activity

This request lists the file activity of a user. Files activity includes the files created, deleted, and restored by the user.

URI

/fss/public/activity/files

HTTP Method

POST

Parameters

The activity/files endpoint requires only request body parameters. The request body must be in JSON format.

Request body parameters:

•activityType (required): Enum

The type of events that are returned:

oMODIFY — Returns events in which files were created, modified, or deleted.

oREAD_LINK — Returns events in which files were accessed through a shared link.

oREAD — Returns events in which files were accessed, but not through a link.

•language (optional): Enum

The language of the events. The default language is English. For this parameter, you can enter:

oen-US — English

ozh-CN — Simplified Chinese

ofr-CA — French Canadian

oja — Japanese

oko — Korean

oes — Spanish (Latin America)

ode — German

•fsId (conditional): String

The unique identifying string of the given user's filesystem. You can acquire the fsId of a user's filesystem using a user/filesystems/list request.

You need to supply an fsId in the first activity/files request for a given user (or if a pageToken is not supplied in the request). For each successful activity/files request, a pageToken is returned. The pageToken identifies the user's filesystem.

If a pageToken is supplied in the request body, you do not need to include an fsId.

•pageSize (optional) : Integer

The maximum number of items to return in the request. The default is 100.

•pageAction (optional): Enum

The portion of the events to return:

o FIRST — Returns the newest events. For example, if your pageSize is 3, then the newest three events are listed.

oLAST — Returns the oldest available events. For example, if your pageSize is 3, then the oldest three events are listed.

oNEXT — Returns the next set of events based on your position in the list of events. Your position in the list of events is determined by the page token. A page token is returned in response to a successful request.

oPREV — Returns the previous set of events based on your position in the events. Your position in the list of events is determined by the page token. A page token is returned in response to a successful request.

Note: If a pageAction and pageToken are not included in the request, the newest set of events are returned.

•pageToken (optional) : String

The page token returned in the body of the previous response. The page token identifies where in the list of events you are.

Example Request

This section contains an example of an activity/collaboration request in cURL. In this request, the access token is "abc=". In an actual request, the access token is much longer.

curl -kiX POST https://anywhere.example.com/fss/public/activity/files -H "Authorization: Bearer abc=" -H "Content-Type: application/json" -H "Accept: application/json" -H "X-HCPAW-FSS-API-VERSION: 4.0.0" -d '{"activityType": "MODIFY", "fsId": "144115188084244481", "pageSize": 2, "pageAction": "FIRST"}'

Description of Response

This section describes the contents of an activity/files response.

Response content:

•pageToken (required) : String

A page token is returned in every response and identifies where in the list of events you are. You can supply this page token in the body of a request to return the next set of events.

You can supply this page token in the body of subsequent requests.

•activites (required): Object

The list of events:

oentry (required): Object

Information about the file or folder that the event is about:

–type (required) : Enum

The type of item (FILE or FOLDER).

–parent (required) : String

The path to the folder that contains the item.

–name (required) : String

The name of the item.

–changeTime (required) : Long

The time at which the item was last accessed or modified. This time is returned in Unix format.

–size (conditional) : Long

If the item is a file, the size of the file in bytes.

–hash (conditional) : String

If the item is a file, the hex encoded sha-384 hash of the file.

–access (required) : Enum

The type of operations a user can perform on the item:

•NO ACCESS — The user cannot access the file or folder.

•VIEWER — The user can view and download the file or folder. However, if the entry is a folder, the user cannot upload files to the folder.

•COLLABORATOR — The user can download, upload, move, and delete files and folders.

–etag (required) : String

The etag of the item. The etag identifies the version of the file or folder and is supplied as a parameter in some of the other File Sync and Share API endpoints.

–state (required) : Enum

The state of the file (CREATE or DELETE).

–sharing (conditional): Object

If the item is a shared folder, team folder, or mobilized share or within one of these, the additional shared folder information:

•canUserManage: Boolean

If true, the user is the Owner of the shared folder or the Manager of the team folder.

•isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. Shared folders, team folders, and mobilized shares are always mount points although the items within them are not.

•label (conditional) : String

If the item is a shared or team folder, the label of the folder.

•type: Enum

The type of collaborative folder (SHARED, TEAM, or MOBILIZED).

•teamFolder (conditional): Object

The additional settings that apply to team folders:

ocontact: String

The contact of the team folder.

oquota: Long

The total storage capacity of the team folder. This is returned in bytes.

–Backup (conditional): Object

If the item is a backup file or folder, the additional backup information:

•isActive: Boolean

If true, the backup folder is connected to the HCP Anywhere desktop application. 

The backup folder can be disconnected from the desktop application if the desktop application is deregistered or if the backup feature is disabled by the system administrator.

•isMountPoint: Boolean

If true, the item is a mount point — also known as mounted filesystem — within the HCP Anywhere filesystem. The backup parent folder, which is automatically created when the desktop application is installed with backup enabled, is always a mount point.

•operatingSystem: Enum

The type of device on which the desktop application is installed (WINDOWS, WINDOWS NSC, or MAC). For version 3.0 or later of the Windows desktop application, the operatingSystem is "WINDOWS NSC".

•hasAssociatedClient: Boolean

Each backup file or folder is associated with the desktop application that performed its backup. If true, the desktop application associated with the backup folder is currently registered. If false, the desktop application associated with the backup folder is currently deregistered.

Note: An application can be deregistered if:

•You deregister it in the User Portal in the My Devices window

• Your administrator deregisters it in the HCP Anywhere Management Console

omsg (required): String

Describes the event.

otimestamp (required): Long

The time at which the event occurred. This time is returned in Unix format.

oisPruned (required): Boolean

If true, the file has been pruned from the HCP Anywhere system. A pruned file is different from a deleted file. When a user deletes a file, the file remains in the HCP Anywhere system until it is pruned.  When pruned, the file is deleted from the HCP Anywhere system and can no longer be recovered. The amount of time the file remains in the system before it is pruned is configured by the HCP Anywhere administrator.
